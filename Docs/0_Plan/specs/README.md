# Plan Specs

本目录用于存放 `0_Plan` 体系下的**设计稿 / 边界 spec / 方案收口文档**。

它负责回答：

- 某个方向为什么要这样设计
- 边界和取舍是什么
- 进入 implementation plan 之前已经达成了哪些共识

它不负责替代：

- `Project_Plan.md` 的项目状态入口职责
- `ongoing/` 中 implementation plan 的执行拆分职责
- `complete/` 中已完成专项的归档职责

## 使用规则

- brainstorming 或其他方案收口流程产出的 spec，默认放进本目录
- spec 获得确认后，再进入 `Docs/0_Plan/ongoing/` 写 implementation plan
- 若 spec 已失效，不应继续作为现役依据，应按实际情况迁移或归档
