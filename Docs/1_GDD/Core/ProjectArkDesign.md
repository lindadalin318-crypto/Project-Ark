# 1. 游戏概述

## 1.1 高概念 (High Concept)

| 维度 | 描述 |
|------|------|
| **类型** | Top-Down 2D 动作冒险，融合银河恶魔城探索 + 类魂叙事氛围 |
| **平台** | PC (Unity 6 + URP 2D) |
| **视角** | 俯视角 (Top-Down View) |
| **操控** | 双摇杆射击 (Twin-Stick Shooter) |
| **一句话梗概** | 在"大静默"吞噬宇宙的终焉时刻，驾驶金丝雀号飞船，通过"星图编织系统"定制武器，在异类星球上探索、战斗、理解——**理解即武器** |

## 1.2 核心体验支柱

| 支柱 | 设计目标 | 关键系统 |
|------|----------|----------|
| **手感优先** | 射击清脆、移动丝滑、后坐力有物理量感 | 3C 系统 + HitStop + 屏幕震动 |
| **Build 多样性** | 玩家通过星图部件的自由组合定义个人战斗风格 | 星图编织系统 (4类部件 × 组合爆炸) |
| **可读的战斗** | 敌人行为有清晰的信号-窗口节奏，死亡是"我没读对"而非 AI 作弊 | 敌人 HFSM + 导演令牌系统 |
| **叙事驱动探索** | 每颗星球是一个哲学实验场，探索驱动力来自"理解异类文明" | 关卡系统 + NPC 互动 + 道德选择 |
| **克制的美学** | 拒绝满屏光污染，追求"晶莹、极简、高爆发"的视觉调性 | 弹道合并规则 + 极简特效分级 |

# 2. 核心玩法循环

## 2.1 战斗核心循环 (30秒循环)

```
移动走位 → 瞄准 → 开火(消耗热量) → 读取敌人信号 → 闪避/反击 → 拾取掉落 → 循环
```

**关键约束**：
- 开火消耗热量，过热 = 3秒无法攻击/冲刺，近乎致命
- 后坐力影响飞船运动轨迹，移动与射击深度耦合
- 敌人攻击遵循"信号-窗口"模型，玩家需识别前摇→闪避→抓恢复期输出

## 2.2 探索循环 (5分钟循环)

```
进入新房间 → 战斗/解谜 → 拾取部件/钥匙 → 发现新路径/被锁住 → 回溯寻找钥匙 → 解锁新区域
```

## 2.3 成长循环 (20分钟循环)

```
探索获得新星图部件 → 回到安全区调整 Build → 以新 Build 挑战新区域 → 
击败 Boss 解锁新槽位/新部件类型 → Build 复杂度提升
```

## 2.4 叙事循环 (贯穿全程)

```
遭遇异类种族 → 无法理解(敌对/误解) → 通过探索/战斗收集信息 → 
理解其逻辑 → 做出道德选择 → 招募/放弃 → 影响最终结局
```

# 3. 飞船操控系统 (3C)

## 3.1 功能概述

| 功能 | 描述 | 目的 |
|------|------|------|
| **移动** | WASD 控制飞船在 2D 平面上移动，有加速/减速惯性 | 冰面滑行手感，移动本身有操作深度 |
| **瞄准** | 鼠标/右摇杆控制飞船朝向（船头追踪光标） | 射击方向由船头决定，非鼠标方向 |
| **冲刺** | 快速位移（i-frame 无敌帧），消耗冲刺冷却 | 核心闪避手段，类魂式规避窗口 |
| **射击** | 按住鼠标左键持续连发，子弹沿船头发射 | 核心输出手段 |
| **后坐力** | 每次开火给飞船施加反向推力 | 射击影响移动轨迹，形成"移动-射击三角关系" |

## 3.2 关键参数

| 参数 | 说明 |
|------|------|
| `MoveSpeed` | 飞船基础移动速度 |
| `Drag` | 移动阻力/惯性感 |
| `RotationSpeed` | 船头追踪光标的转速 |
| `DashDistance` / `DashDuration` | 冲刺距离和时长 |
| `DashCooldown` | 冲刺冷却时间 |
| `RecoilForce` | 开火后坐力强度 |

## 3.3 设计意图

**"移动即战斗"**——通过后坐力和光帆系统，飞船的运动状态直接影响战斗输出。玩家不只是在"站桩射击"，而是在"用飞行轨迹作画"。这与光帆（如多普勒光帆：速度越快伤害越高）深度绑定，形成独特的操作流派。

# 4. 星图编织系统

## 4.1 功能概述

**星图编织系统**是玩家构建飞船战斗方式的模块化武器系统。  
**核心隐喻**：玩家在精密星盘上镶嵌宝石——星核是光源，棱镜是滤镜，光源透过滤镜折射出最终的攻击形态。

| 维度 | 描述 |
|------|------|
| **功能** | 替代传统"武器切换"，采用"并行轨道 + 垂直修正"的组装逻辑 |
| **目的** | 提供指数级组合深度，让每个玩家都能找到独特的战斗风格 |
| **交互方式** | 容器逻辑——将部件放入对应槽位即自动生效，无需连线或编程 |
| **体验目标** | 易上手（扔进去就行）、有深度（空间管理 + 热量博弈）、克制优雅 |

## 4.2 星图结构：三明治双轨道

```
┌─────────────────────────────────────────────────────┐
│  主星轨 (PRIMARY) — 鼠标左键                          │
│  ┌─────────────────────────────────────────────┐    │
│  │ 上层：棱镜槽 [  ][  ][  ] (容量: 3格)         │    │
│  │              ↓↓↓ (垂直注入)                   │    │
│  │ 下层：星核槽 [  ][  ][  ] (容量: 3格)         │    │
│  └─────────────────────────────────────────────┘    │
│                                                       │
│  副星轨 (SECONDARY) — 鼠标右键                        │
│  ┌─────────────────────────────────────────────┐    │
│  │ 上层：棱镜槽 [  ][  ][  ] (容量: 3格)         │    │
│  │              ↓↓↓ (垂直注入)                   │    │
│  │ 下层：星核槽 [  ][  ][  ] (容量: 3格)         │    │
│  └─────────────────────────────────────────────┘    │
│                                                       │
│  光帆插槽 (引擎位) — 被动生效                          │
│  伴星轨道 (外环)   — 自动触发                          │
└─────────────────────────────────────────────────────┘
```

**发射逻辑（齐射 Alpha Strike）**：按下开火键 → 下层所有星核在同一帧同时发射 → 上层所有棱镜效果平均注入每个星核。

## 4.3 四大部件类型

### 4.3.1 星核 (Star Core) — 发射源

**功能**：决定"射出的是什么"。位于星轨下层。

| 家族 | 代表部件 | 机制特征 | 手感关键词 |
|------|----------|----------|------------|
| **实相系 (Matter)** | 撕裂者(机枪)、风暴散射(霰弹)、转管炮、死星主炮 | 实体弹头，有碰撞和后坐力 | 撞击、重量感 |
| **光谱系 (Light)** | 聚焦透镜、棱光射线、裂变光束 | 射线判定，瞬间命中，持续照射 | 精准、穿透 |
| **波动系 (Echo)** | 震荡环、脉冲新星、余震 | 膨胀碰撞体，范围判定，可穿墙 | AOE、震慑 |
| **异象系 (Anomaly)** | 回旋镖、布雷器 | 复杂运动轨迹，独立行为逻辑 | 陷阱、诡计 |

**当前内容量**：12 个星核（4 家族各 2-4 个），含 Size 1-3 的部件。

### 4.3.2 棱镜 (Prism) — 修正器

**功能**：决定"子弹形态如何改变"。位于星轨上层。  
**生效逻辑**：垂直注入——棱镜效果平均分配给下方所有星核。

| 家族 | 代表部件 | 修改逻辑 |
|------|----------|----------|
| **分形 (Fractal)** | 双生(+2弹)、弹幕(+5弹)、三连发、齐射 | 修改发射数量和散射角 |
| **流变 (Rheology)** | 极速、巨像、连射、重弹、远射、反弹、轻载 | 修改数值属性(速度/伤害/射程/热量) |
| **晕染 (Tint)** | 霜冻、灼烧、腐蚀 | 附加状态效果(减速/DOT/降防) |

**当前内容量**：17 个棱镜（3 家族各 3-8 个）。

### 4.3.3 光帆 (Light Sail) — 驾驶风格

**功能**：将飞船运动状态转化为战斗增益。独立引擎插槽（仅 1 格）。  
**设计目的**：定义玩家流派——冲刺流、近身流、站桩流、操作流。

| 光帆 | 触发条件 | 效果 |
|------|----------|------|
| 多普勒光帆 | 监听飞船速度 | 速度越快，伤害越高 (+5%/单位速度) |
| 虚空引擎 | 停止开火 3秒 | 下次攻击必定暴击 |
| 擦弹引擎 | 极限闪避事件 | 每次闪避 +10% 射速 |
| 稳态力场 | 快速旋转 | 热量散发速度翻倍 |
| 重装帆 | 站桩 >1秒 | 逐秒叠加减伤，无法冲刺 |
| 脉冲帆 | 冲刺 | 冲刺路径造成 25 点伤害 |

**当前内容量**：8 个光帆。

### 4.3.4 伴星 (Satellite) — 自动化模组

**功能**：悬浮在飞船周围的自动化逻辑。逻辑模式：IF (条件) THEN (行为)。

| 伴星 | 触发条件 | 自动行为 |
|------|----------|----------|
| 复仇之月 | 护盾破碎 | 释放冲击波 |
| 清道夫 | 拾取掉落物 | 冷却 50 热量 |
| 双子座 | 右键开火 | 自动向后方发射左键武器 |
| 紧急制动 | 生命 <20% | 获得 2 秒无敌 |
| 自动机炮 | 每 1.5 秒 | 自动射击最近敌人 |
| 拾取磁铁 | 持续 | 吸引 200 范围内掉落物 |

**当前内容量**：6 个伴星。

## 4.4 空间管理：大部件博弈

| 部件尺寸 | 槽位占用 | 设计公式 |
|----------|----------|----------|
| Size 1 | 1 格 | 灵活、可组合、低单发伤害 |
| Size 2 | 2 格 | 中等集成度 |
| Size 3 | 3 格(满槽) | 极高单发伤害、低热量效率、零组合自由度 |

> **大体积 = 高集成度 = 更低的热量消耗 + 更高的单发伤害 − 较低的组合自由度**

## 4.5 预设 Build 示例

| Build 名 | 光帆 | 主武器配置 | 打法 |
|----------|------|------------|------|
| **星际舞者** | 多普勒(速度=伤害) | 散弹 + 加速棱镜 | 冲刺最高速瞬间开火，一击脱离 |
| **静默刺客** | 虚空引擎(停手隐形) | 激光 + 聚光透镜 | 隐形走位→显形一发狙击→逃跑 |
| **几何风暴** | 稳态力场(转圈散热) | 机枪 + 分裂棱镜 + 清道夫伴星 | 冲进怪堆疯转，靠拾取维持无限火力 |

## 4.6 发射管线流程

```
Step 1: 热量检查 → 过热/冷却中则拒绝开火
Step 2: 数据快照 → 读取当前星轨的所有星核 + 棱镜
Step 3: 管道加工 → 棱镜修正注入星核数据（SnapshotBuilder.Build()）
Step 4: 实例化   → 对象池生成子弹，按家族分发（Matter/Light/Echo/Anomaly）
Step 5: 结算反馈 → 累加热量 + 判断过热 + 音效 + 后坐力
```

# 5. 热量系统

## 5.1 功能概述

| 维度 | 描述 |
|------|------|
| **功能** | 替代传统弹药/蓝条的资源限制机制 |
| **世界观包装** | 热量 = 飞船逻辑框架与异类逻辑的不兼容度（非字面温度） |
| **目的** | 限制无限堆叠、创造"重炮 vs 机枪"的流派分化、赋予战斗节奏感 |

## 5.2 核心公式

```
单次射击产热 = Σ(星核产热) + Σ(棱镜产热)
射击间隔    = 单次射击产热 / 飞船散热速度
```

## 5.3 流派分化

| 流派 | 产热特征 | 射击间隔 | 操作感 |
|------|----------|----------|--------|
| **重炮流** | 极高产热 | ~2秒/发 | 精准瞄准、一发入魂 |
| **机枪流** | 极低产热 | ~0.1秒/发 | 按住不放、持续压制 |

## 5.4 过热惩罚

- **触发条件**：热量条达到最大值
- **惩罚内容**：进入**"静默冷却"**状态，3 秒内无法攻击、光帆失效（无法冲刺）
- **设计意图**：过热 ≈ 战斗中的"死刑判决"，迫使玩家主动管理热量而非无脑输出

## 5.5 主动散热机制

| 机制 | 触发方式 | 效果 |
|------|----------|------|
| 被动散热 | 自动 | 飞船每秒减少固定热量 |
| 完美闪避散热 | 冲刺时极限闪避弹幕 | 大幅降低热量（高风险高回报） |
| 清道夫伴星 | 拾取掉落物 | 冷却 50 热量 |
| 轻载棱镜 | 装备 | 整体热量消耗 -30% |

## 5.6 叙事整合

**"战斗中才能散热"**——在静默区（安全/探索区域），热量只增不减，每一发子弹都是有限资源；在嘈杂区（战斗区域），战斗本身就是散热。这创造了一个反直觉但深刻的循环：**平静比战斗更危险**。完美契合"静默方舟"的主题。

# 6. 敌人 AI 系统

## 6.1 功能概述

| 维度 | 描述 |
|------|------|
| **功能** | 提供有深度、可读、可调的敌人行为系统 |
| **目的** | 让战斗成为"玩家与规律的博弈"而非"与随机数的对抗" |
| **设计哲学** | 精准的演员，而非超级大脑——敌人不需要"聪明"，需要"可读" |

## 6.2 三层架构

```
┌────────────────────────────────────────────────────┐
│            导演层 (EnemyDirector)                     │
│  攻击令牌池 · 全局协调 · 群体调度                      │
└───────────────────────┬────────────────────────────┘
                        │ 协调
┌───────────────────────▼────────────────────────────┐
│          大脑层 (EnemyBrain + HFSM)                  │
│  外层状态机：Idle → Chase → Engage → Return          │
│  内层状态机：Telegraph → Attack → Recovery           │
│  感知系统：视觉锥 + 听觉事件 + 记忆衰减               │
└───────────────────────┬────────────────────────────┘
                        │ 指令
┌───────────────────────▼────────────────────────────┐
│          躯壳层 (EnemyEntity)                        │
│  移动执行 · HP/韧性 · 受击反馈 · 对象池 · IDamageable │
└────────────────────────────────────────────────────┘
```

## 6.3 信号-窗口攻击模型

这是类魂战斗的灵魂——每个敌人攻击都有清晰的阶段划分：

| 阶段 | 时长(可配) | 敌人行为 | 玩家策略 |
|------|------------|----------|----------|
| **前摇 (Telegraph)** | 0.3-1.0s | 停止移动、蓄力变红 | 识别攻击类型→准备闪避 |
| **出招 (Attack)** | 0.1-0.5s | 释放伤害，不可转向(Commitment) | 利用走位躲避 |
| **恢复 (Recovery)** | 0.5-1.5s | 硬直不能动 | **反击窗口**，最大化输出 |

**难度调节方式**：缩短前摇 = 更难读取；延长恢复 = 更大反击窗口。

## 6.4 四大敌人原型

| 原型 | AI 行为 | 战术定位 | 给玩家的压力 |
|------|---------|----------|-------------|
| **莽夫 (Rusher)** | 发现→直线冲锋→近战攻击→硬直 | 近身压迫、数量取胜 | 迫使玩家保持移动 |
| **射手 (Shooter)** | 保持理想射距→三连发→被近身就后撤 | 远程骚扰、迫使切后排 | 迫使玩家分配注意力 |
| **刺客 (Stalker)** | 半透明潜行→绕到背后→极快背刺→远离 | 突袭、惩罚不注意后方 | 迫使玩家警惕背后 |
| **炮台 (Turret)** | 不移动→旋转锁定→蓄力→激光/蓄力弹 | 区域封锁、限制走位空间 | 迫使玩家规划路径 |

## 6.5 导演系统 (攻击令牌)

**问题**：多敌人同时攻击 = 被围殴 = 体验灾难  
**解决方案**：全局令牌池限制同时攻击的敌人数量（默认最多 2 个），其余敌人进入 **Orbit 环绕状态**（吼叫助威、在玩家周围游弋等待）。

**效果**：营造"电影感的轮流单挑"节奏——每次只面对有限的攻击压力，但威胁持续不断。

## 6.6 感知系统

敌人不是全知全能的，感知系统赋予它们"五感"并为玩家提供策略空间：

| 感知层 | 机制 | 玩家策略空间 |
|--------|------|-------------|
| **视觉锥** | 扇形 LoS 检测 (5Hz) | 利用掩体/盲区绕过敌人 |
| **听觉** | 玩家开火产生噪音，范围内敌人被激活 | 使用低噪音武器避免惊动远处敌人 |
| **记忆衰减** | 失去视线后维持记忆一段时间，超时后脱战回归 | 脱离追击→等记忆消失→安全通过 |

## 6.7 进阶系统

| 系统 | 功能 | 目的 |
|------|------|------|
| **恐惧系统** | 同伴死亡/韧性击破→累积恐惧→超阈值逃跑 | 营造"势如破竹"的爽感，奖励激进玩家 |
| **阵营系统** | 不同 FactionID 的敌人互相攻击 | "驱虎吞狼"策略空间 |
| **闪避/格挡 AI** | 检测来袭弹头→侧闪/举盾 | 提升精英敌人的威胁感和策略需求 |
| **精英词缀** | 随机附加增益：狂暴/护盾/爆炸/吸血/反伤 | 低成本提升内容多样性 |
| **多阶段 Boss** | HP 阈值触发阶段切换，无敌过渡，更换攻击模式 | Boss 战的层次感和叙事承载 |

## 6.8 敌人规划 (Plus 版)

|------|------|------|

| 特化 - 机动型 | ~8 种 | 光帆质检员，迫使学习高级移动 |

# 7. 关卡系统

## 7.1 功能概述

| 维度 | 描述 |
|------|------|
| **功能** | 提供银河恶魔城式的互联房间网络，支持探索、回溯、锁钥解谜 |
| **目的** | 承载战斗遭遇、叙事事件、玩家成长的空间载体 |
| **技术方案** | 单场景 + Tilemap 房间 + Cinemachine Confiner |

## 7.2 世界结构

```
星球 (Planet) — 每个星球 = 一个 Unity Scene
├── 区域 (Zone) — 由房间组成的主题区块
│   ├── 房间 (Room) — 基本探索单元 (20-40 间/星球)
│   │   ├── 普通房间：预置敌人，可进可出
│   │   ├── 竞技场：进入锁门→波次战斗→全清开门
│   │   ├── Boss 房间：Boss 战专用竞技场
│   │   └── 安全区：检查点、NPC、补给
│   └── 门/通道 (Door) — 连接房间，支持锁钥条件
└── 楼层 (FloorLevel) — 垂直多层结构
```

## 7.3 房间系统

| 功能 | 描述 | 目的 |
|------|------|------|
| **房间边界** | BoxCollider2D Trigger 定义范围 | 玩家进入时激活房间内容 |
| **Cinemachine Confiner** | 镜头约束在当前房间边界内 | 战斗空间感清晰 |
| **敌人激活/休眠** | 只有当前房间的敌人活跃 | 性能优化 + 空间隔离 |
| **房间状态** | 未发现/已进入/已清理/已锁定 | 地图标记和重访逻辑 |
| **多房间变体** | 同一房间可随世界阶段切换不同配置 | 动态世界感 |

## 7.4 门与通道

| 门状态 | 触发条件 | 典型场景 |
|--------|----------|----------|
| `Open` | 无条件 | 普通通道 |
| `Locked_Combat` | 清除房间内所有敌人 | 竞技场门 |
| `Locked_Key` | 持有对应钥匙道具 | 银河恶魔城回溯解锁 |
| `Locked_Ability` | 持有特定光帆/能力 | 能力门 |
| `Locked_Schedule` | 世界时钟特定阶段 | 定时开放的隐藏通道 |

**过渡演出**：玩家走到门口 → 短暂淡黑 → 传送到目标房间入口点 → 淡入。层间过渡使用更长淡黑 + 特效粒子。

## 7.5 多层结构

采用**混合方案**：99% 用淡黑传送（简洁可靠），极少数叙事关键时刻用无缝掉落演出（震撼瞬间）。

- 不同楼层的房间在同一 Scene 中但物理分离（Y 偏移 -500 units）
- 每一层可有完全不同的 Tilemap 主题/色调/音乐
- 对 RoomManager 来说，层间传送和普通过门完全一样

## 7.6 遭遇系统

**波次战斗 (WaveSpawnStrategy)**：

```
进入竞技场 → 门锁死 → 警报音效 →
Wave 1: 3×Crawler (基础) →
Wave 2: 2×Drone + 1×Loader (组合压力) →
Wave 3: 1×Elite + 2×Crawler (精英+杂兵) →
全清 → 门解锁 + 掉落奖励
```

- 波次配置通过 `EncounterSO` 数据驱动
- 支持波次间延迟、动态难度调整
- 竞技场通关后不重置（持久状态）

## 7.7 世界时钟与动态关卡

### 大框架：事件驱动阶段（不可逆进度）

```
阶段 1: Boss 存活期 → 击败 Boss → 阶段 2: 新区域永久开放 → 获得关键道具 → 阶段 3 ...
```

### 小周期：轻量时间循环（15-20 分钟一个周期）

| 时段 | 效果 |
|------|------|
| **辐射潮** | 某些区域有环境伤害，NPC 躲藏，特殊敌人出没 |
| **平静期** | NPC 交易，定时门打开，最佳探索窗口 |
| **风暴期** | 敌人增强，特殊掉落，视听氛围变化 |
| **寂静时** | 低能见度，隐藏通道出现，稀有 NPC |

**设计目的**：让世界感觉是"活的"——玩家可以学习这些规律来优化探索策略。

## 7.8 环境系统

| 系统 | 功能 | 实现方式 |
|------|------|----------|
| **环境危害 (Hazard)** | 非战斗持续压力（酸液/激光栅栏/地雷） | DamagePayload 通过统一伤害管线造伤 |
| **环境互动** | 射击特定物体产生效果（音叉树、克拉德尼花） | 每星球独特的环境交互系统 |
| **涟漪工具** | 独立探索工具，非武器（解密/推物/显形/净化） | 长按释放声波脉冲 |

# 8. 进度与成长系统

## 8.1 功能概述

| 维度 | 描述 |
|------|------|
| **功能** | 管理玩家在游戏中的横向（部件收集）和纵向（槽位解锁）成长 |
| **目的** | 提供持续的"变强感"和"发现新可能性"的驱动力 |
| **设计哲学** | 成长不是数值膨胀，而是"可能性的展开" |

## 8.2 星图槽位解锁（纵向成长）

| 阶段 | 主星轨 | 副星轨 | 光帆 | 伴星 | 解锁条件 |
|------|--------|--------|------|------|----------|
| **初始** | 1棱镜 + 1星核 | 锁死 | 0 | 0 | 游戏开始 |
| **前期** | 2棱镜 + 2星核 | 1棱镜 + 1星核 | 1 | 1 | 守夜人修复 |
| **中期** | 3棱镜 + 3星核 | 2棱镜 + 2星核 | 1 | 2 | 剧情推进 |
| **后期** | 解锁大体积槽 | 3棱镜 + 3星核 | 2 | 3 | 完全体 |

## 8.3 部件获取（横向成长）

| 获取方式 | 掉落物 | 用途 |
|----------|--------|------|
| **暴力击杀** (不装共情协议) | 熵值/废料 | 通用货币：扩充背包、升级基础属性 |
| **理解击杀** (装共情协议) | 逻辑图谱 | 解锁新部件（新星核/棱镜/光帆） |

**共情协议**是游戏叙事与玩法结合的核心模块：
- 伤害修正 -40~80%（降低输出以"理解"敌人）
- 子弹变为柔和解析色，命中施加"解析中"状态
- 敌人在"解析中"状态下死亡 = 理解成功 = 掉落逻辑图谱
- **设计目的**：为了变强（获得新能力），你必须让自己变弱——极佳的风险博弈

## 8.4 锁钥系统

| 锁类型 | 钥匙来源 | 典型场景 |
|--------|----------|----------|
| 彩色钥匙卡 | 探索拾取 | 基础的区域解锁 |
| Boss 掉落钥匙 | 击败 Boss | 开启通往新大区域的路径 |
| 能力门 | 获得特定光帆/能力 | 回溯解锁之前无法到达的区域 |

## 8.5 检查点系统

- 散布在关卡各处的固定激活点
- 交互激活：恢复 HP + 恢复热量 + 设置重生点 + 自动存档
- 死亡后从最后激活的检查点重生
- 已通关的竞技场/Boss 房间不重置，未清理的房间敌人重置

## 8.6 死亡与重生

```
玩家死亡 → 
  死亡演出（PrimeTween 淡黑 + 音效）→ 
  重置当前房间敌人 → 
  在最后检查点重生 → 
  自动存档
```

## 8.7 存档系统

已有 `SaveManager + PlayerSaveData` 支持：
- 已访问房间 ID 列表
- 已击败 Boss ID 列表
- 进度标记 (Flags)
- 最后检查点 ID
- 世界时钟时间 / 世界阶段

### 动态格子扩展（混合方案）

随游戏进度解锁更多格子时：先横向扩展列数(→8)→缩小槽位尺寸(64→40px)→换行。

# 11. 视听表现

## 11.1 视觉风格

| 维度 | 规范 |
|------|------|
| **整体调性** | 认知陌生化——异类种族不是"带触手的人"，而是视觉上挑战认知的存在 |
| **色彩心理** | 安全=冷青色，危险=赤红色，解析/理解=淡金色，大静默=去色/黑白 |
| **环境叙事密度** | 废墟中充满碎片和痕迹，避免"塑料感"的过于干净 |
| **弹道视觉** | 极简主义——普攻半透明低调，副武器和 Boss 大招允许全屏特效 |
| **弹道合并** | 单帧 20+ 同类子弹→合并为 1 个带拖尾的高亮光球 |

## 11.2 大静默视觉表现

| 阶段 | 视觉 |
|------|------|
| **语义剥离** | UI 出现乱码，对话选项模糊 |
| **物理崩塌** | 物体边缘锯齿，星空出现裂痕 |
| **绝对寂静** | 画面去色至黑白，维度坍缩感 |

## 11.3 听觉架构

| 层级 | 设计 |
|------|------|
| **物理传播** | 超导声场——高频保留完整，远处钟声如同耳边 |
| **材质拟音** | 玻璃珠摩擦(克拉德尼沙原)、有音高的撞击声(管风琴遗迹) |
| **静默区** | 沉闷、低保真、无混响——极度幽闭恐惧感 |
| **敌人发声** | 无声带——巡逻时正弦波嗡嗡，攻击时锯齿波刮擦，受伤时乐器碎裂 |
| **玩家飞船** | 粗糙、颗粒感、工业化——与纯净世界形成"脏 vs 净"对比 |

## 11.4 互动配乐系统

| 状态 | 配乐层 |
|------|--------|
| **探索** | 稀疏玻璃琴 + 风声（孤独感）|
| **战斗** | 叠加管风琴 + 高 BPM 电子鼓点 |
| **特殊机制** | 侧链压缩——玩家开火时背景音乐瞬间压低，枪声成为"主节奏" |
| **编织态** | 800Hz 低通滤波器闷住外界声音 + 晶体嗡嗡环境音 |

## 11.5 音效反馈优先级

