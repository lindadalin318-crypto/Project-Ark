# 项目代号：《魔王的人力资源部》 (Demon HR: Hero Ruination) — V4.2 雾灯情报版

## 版本变更摘要 (V2.1 → V3.0 → V3.1 → V4.0 → V4.1 → V4.2)

V3.0 保留了 V2.1 的所有核心设定（战役季度制、群聊主战场、社交节点操纵、中间胜态链、招安路线、角色持续化），并针对 V2.1 评审中暴露的四个结构性问题进行了系统性补强：

| 问题 | V2.1 状态 | V3.0 解决方案 |
|------|-----------|--------------|
| 操作之间缺乏组合性 | 5 种操作各自独立 | 新增 **操作链系统 (Operation Chain)**：引入"布置→触发→放大"三拍节奏，卡牌之间产生乘法 synergy |
| Admin 权限无代价 | 每次操作必定成功，无风险 | 新增 **暴露风险系统 (Exposure Risk)**：每次操作积累可疑度，勇者方会升级反制 |
| 全知视角缺乏博弈 | 玩家默认看到所有信息 | 新增 **主动情报系统 (Active Intel)**：情报带宽有限，玩家必须选择监听什么 |
| 右侧归因模糊 | 2-5 秒自动战斗难以归因 | 重设计右侧为 **事故复盘系统 (Incident Replay)**：先给结果，再可选展开因果链 |

V3.1 在 V3.0 基础上新增两个重大设计模块：

| 新增模块 | 解决的问题 | 核心内容 |
|----------|-----------|---------|
| **MICE 情报官操作体系** | V3.0 招安系统深度不足，策反过程缺少经营感 | 引入真实情报学的 MICE 四维策反模型（金钱/理念/胁迫/自尊）+ 资产发展周期（发现→评估→培养→策反→运营） |
| **双面立场系统 (Dual Stance)** | 系统只提供破坏工具会把角色框定为"待攻破的目标"，阻碍共情 | 同一套卡牌可反向使用（保护而非破坏），引入魔王忠诚度计量器，让"保护谁、牺牲谁"成为与破坏同等深度的策略线 |

V4.0 针对 V3.1 评审中暴露的两个结构性问题进行了系统性治理：

| 问题 | V3.1 状态 | V4.0 解决方案 |
|------|-----------|--------------|
| 10+ 决策轴同时争夺注意力 | 每回合玩家需同时关注情报带宽、出牌、操作链、暴露、忠诚度、MICE、资产维护、凝聚力、渗透者指令、汇报节点 | 新增 **注意力分层架构 (Attention Triage)**：将所有决策轴分为前台 / 告警 / 被动三层，每回合主动决策压缩至 3 个（V4.1 进一步将情报带宽与出牌预算合并为统一行动力） |
| 渗透者系统每回合需逐条下达指令 | 渗透者的 4 种能力（发言/私聊/传递情报/破坏）需要每回合手动操作 | 渗透者改为 **策略设定模式**：设定行为倾向后自动执行，仅在关键分叉弹出确认 |
| 汇报节点定义模糊 | 汇报像是回合内的一个额外行动 | 汇报节点改为 **地图事件节点**：在小地图上作为固定遭遇出现，有自己的专属 UI 和决策空间 |
| 文本质量是致命依赖且无解 | "混合方案"一笔带过，无具体管线和工作量估算 | 新增 **文本生产管线（五层防线架构）**：设计减量→消息DNA→编排引擎→手写锚点→验证管线，总工作量 6-8 周 |

V4.1 针对 V4.0 评审中暴露的资源经济问题进行了系统性重构：

| 问题 | V4.0 状态 | V4.1 解决方案 |
|------|-----------|--------------|
| 预算与带宽各自独立，缺乏"经营感" | 每回合预算 3 + 带宽 3 = 6 点分属两个不互通的池 | 合并为统一的 **行动力 (Action Points)**：3 点/回合，监听和出牌竞争同一资源池，制造"看 vs 做"核心取舍 |
| 无季度级资源规划 | 所有资源按回合固定刷新，没有中期经营维度 | 新增 **经费储备 (Reserve Fund)**：季度总量制，可提取到行动力实现爆发回合 |
| 优雅玩法缺乏实质性奖励 | 优雅度仅影响评价和卡牌奖励 | 新增 **暗线网络 (Shadow Network)**：低暴露操作积累"政治资本"，可兑换零暴露操作 |
| 跨季度资源缺乏互动 | 角色档案/魔王信任/公会反制各自独立 | 整合为 **三层资源联动**：魔王信任→下季度经费储备；公会档案→下季度敌方反制；人事档案→情报消耗折扣 |

V4.2 针对 V4.1 情报系统的三个设计隐患进行了系统性完善：

| 问题 | V4.1 状态 | V4.2 解决方案 |
|------|-----------|--------------|
| 花 AP 监听可能什么都看不到——"空包弹"问题 | 未付费的频道显示"加密通讯—内容未知"，付费后可能发现无战术价值 | **雾灯模型 (Fog Lamp)**：所有频道免费展示摘要+热度信号（🔴🟡🟢🔵），AP 解锁的是"精度"而非"访问权"；三种情报货币保证每次投资必有产出 |
| 精心设计的角色深层内容可能永远不被发现 | 角色信息散落在不同系统中，无收集感和发现驱动力 | **角色情报拼图 (Intel Jigsaw)**：每个角色一张可视化拼图面板（12-15 碎片），含公开/情报/时机/深层四类碎片，完成度驱动探索欲 |
| 角色的 MICE 可行性缺乏侦查过程 | MICE 四维雷达直接显示进度，缺少"发现"的过程 | MICE 路线可行/锁死由**拼图碎片**控制：必须通过情报投资或时机窗口才能确认某条路线是否可行，关键路线有 2-3 条冗余发现路径 |

全部系统模块清单：

- **三层资源经济 (Resource Economy)**：行动力（回合级，监听与出牌共池）+ 经费储备/暗线网络（季度级）+ 人事档案/魔王信任/公会档案（跨季度）
- **注意力分层架构 (Attention Triage)**：前台 3 决策 / 告警阈值 / 被动自动化，压缩认知负荷
- **文本生产管线 (Text Production Pipeline)**：五层防线（设计减量 / 消息 DNA / 编排引擎 / 手写锚点 / 验证管线）
- **操作链系统 (Operation Chain)**：布置→触发→放大三拍节奏，卡牌之间产生乘法 synergy
- **暴露风险系统 (Exposure Risk)**：每次操作积累可疑度，勇者方会升级反制
- **主动情报系统 (Active Intel) — 雾灯模型**：免费摘要+热度信号+付费精度，三种情报货币保证无空包弹
- **角色情报拼图 (Intel Jigsaw)**：每角色 12-15 碎片（公开/情报/时机/深层），跨季度持续化，驱动 MICE 发现过程和重玩探索
- **事故复盘系统 (Incident Replay)**：先给结果，再可选展开因果链
- **优雅度评价 (Elegance Rating)**：同一目标有粗暴和精妙两种达成方式
- **连锁反应 / 棋子化 (Cascade & Pawn)**：被操纵的勇者自发产生冲突节点
- **渗透者系统 (Infiltrator)**：Q2+ 解锁，安插卧底；V4.0 改为策略设定模式
- **卡牌体系重设计**：四大家族 × 三个节奏层级 × 双面使用，支持构筑流派与 combo 路线
- **人格交互矩阵 (Personality Matrix)**：人格之间有主动化学反应
- **MICE 情报官操作体系**：四维策反模型 + 资产发展周期 + 策反后运营
- **双面立场系统 (Dual Stance)**：卡牌双面使用 + 魔王忠诚度 + 保护与破坏的双向博弈

**游戏类型**：单机 RPG / 战役式 Deckbuilding / 社交工程模拟 / 多结局  
**视觉风格**：复古像素风 + 现代职场通讯软件 UI 包装  
**核心体验**：潜入勇者通讯后台，通过情报监听、信息操纵与组织手段，从内部瓦解冒险小队——或用同一套工具暗中保护你在乎的人  
**一句话卖点**：你不是打败勇者的人，你是让勇者自己打败自己的人——除非你选择不这么做

## 一、产品定位 (Product Positioning)

本作不是纯《杀戮尖塔》式的高随机爬塔，也不是纯《欺诈之地》式的重剧情对话冒险，而是一个 **固定战役结构 + 每季度局内构筑 + 角色跨季度持续 + 社交工程为核心博弈** 的单机战役型卡牌 RPG。

### 1.1 核心结构定义

- **整个游戏**：一个完整的 `Campaign`（战役）
- **每个季度**：一个固定目标小队的 `Run`（局）
- **每个季度内**：一张小型地图，由多个遭遇、事件、休整点组成

### 1.2 核心卖点

- 玩家不是正面击败勇者，而是作为魔王军 HR 在幕后"做局"
- 操纵的对象不是血量和护甲，而是承诺、信任、压力、权威和信息
- 勇者角色带着创伤、关系与履历跨季度持续存在
- 同一个目标有粗暴和精妙两种达成方式——精妙路径是本作的技巧天花板
- 同一套工具可以用于破坏，也可以用于保护——你的立场才是真正的游戏
- 策反不是"打出一张卡"，而是跨越多个回合的情报培养与心理经营（MICE 四维模型）
- 游戏存在多结局，玩家的立场会在"忠于魔王"与"共情勇者"之间逐步分化

### 1.3 竞品差异化

| 维度 | 杀戮尖塔 | 欺诈之地 | 本作 |
|------|---------|---------|------|
| 核心战斗 | 打怪出牌 | 对话说服 | 社交工程 + 信息操纵 |
| 敌人存在感 | 机制载体 | 有性格的对话对象 | 会自我修复、会反侦察的完整社会组织 |
| 随机性 | 高（每局全新地图） | 中 | 低（固定战役，局内构筑变化） |
| 跨局连续性 | 无 | 低 | 高（角色/关系/创伤/招安跨季度持续） |
| 玩家权力幻想 | 强力战士 | 巧舌如簧的说客 | 幕后操盘的情报主管 |

## 二、战役结构 (Campaign Structure)

### 2.1 总体思路

游戏由 `Q1-Q4`（标准）或 `Q1-Q5`（完整版含终局）组成。每个季度都是一个相对固定的 Run，玩家需要在 HR 资源耗尽前瓦解当前季度负责的小队。

### 2.2 季度原型

- **Q1：初次出征勇者小队**
  - 作用：教学季度，建立"侦察 → 监听 → 干预 → 执行"的四阶段循环
  - 特点：成员年轻、性格外露、关系简单，无反侦察能力
  - 系统解锁：基础五操作（撤回/伪造/延迟/放大/泄露），暴露风险入门
- **Q2：皇家骑士团**
  - 作用：引入反侦察与渗透者机制
  - 特点：可能包含 Q1 存活成员；队伍更团结、领导力更强
  - 系统解锁：渗透者系统，操作链 combo，勇者反制升级（Awareness Level 1）
- **Q3：精英远征队**
  - 作用：将小队矛盾升级为制度矛盾、荣誉矛盾与功绩矛盾
  - 特点：内部派系明确，存在预设裂痕但也有更强的自愈能力
  - 系统解锁：高级行政操作，双卧底，勇者反制升级（Awareness Level 2）
- **Q4：王国直属讨魔队 / 圣征军**
  - 作用：引入王国最强、最稳定、最难瓦解的官方精锐
  - 特点：高凝聚力、高纪律、有专职反间角色
  - 系统解锁：全卡池开放，终极 combo 路线
- **Q5：终局季度（可选）**
  - 作用：根据前几个季度的存活者、招安者与重大事件，正式导向多结局

### 2.3 失败结构

- 若某季度失败，**直接重打该季度**，不回退到前一季度
- 重打时保留已获得的卡牌和权限，但队伍状态重置
- 设计原则：季度是清晰的章节边界，失败惩罚要存在但不能摧毁叙事连续性

### 2.4 季度长度

每个季度为一张小型地图，包含：

- 普通遭遇（3-5 个）
- 固定叙事事件点（1-2 个）
- 休整 / 牌组调整点（1-2 个）
- 季度 Boss 遭遇或终局事件（1 个）

单季度预期游玩时长：45-90 分钟。

## 三、核心视觉与信息流 (The Group Chat Interface)

玩家作为魔王军 HR，拥有勇者公会内部通讯的"后台管理员权限"。

### 3.1 双屏表达

- **左侧主屏（70%）**：勇者小队通讯群聊 + 情报面板
  - 展示群聊消息流、提炼后的社交节点、关系/状态板
  - 这是玩家的**主决策界面**
- **右侧副屏（30%）**：事故复盘终端
  - 不再是"实时自动战斗演出"，而是**结果先行 + 可选因果回放**
  - 核心职责：把左侧操纵的后果以可理解、可复盘的方式呈现

### 3.2 信息展示原则

- 文本负责叙事乐趣与角色塑造
- 图标、标签和状态 Token 负责策略判断
- 玩家平时看摘要信息做决策，想看乐子时再阅读完整群聊
- 左侧是主决策界面，右侧是后果验证与爽感界面

### 3.3 右侧副屏：事故复盘系统 (Incident Replay)

V2.1 的"2-5 秒高速自动战斗"存在归因模糊的根本缺陷。V3.0 重新定义右侧为**事故复盘终端**：

#### 执行阶段的展示流程

1. **结果速报**（0.5 秒）
   - 先直接给出本回合结果摘要：谁受伤了、谁的承诺违约了、谁和谁吵起来了
   - 用状态变化图标一目了然地展示

2. **因果链回放**（可选展开，每条 2-3 秒）
   - 玩家可点击任意事故节点，展开一条 2-3 步的因果链：
   - 例如：`[你撤回了牧师的治疗承诺] → [牧师在执行时犹豫了一拍] → [战士在缺少治疗的窗口期被打到危险血线] → [战士的暴躁人格触发，向牧师发难]`
   - 这条链清晰展示了"玩家操作 → 角色行为改变 → 战斗事故 → 社交后果"的完整因果

3. **关键时刻特写**（自动触发，1-2 秒）
   - 当出现死亡、叛逃、招安机会、团队崩溃等重大事件时，自动播放一段像素风特写动画
   - 配合角色台词气泡，强化戏剧感

#### 设计原则

- **归因清晰优先于视觉表演**：玩家必须能准确知道"这个事故是我的哪步操作导致的"
- **结果先行，过程可选**：不强迫玩家看完整回放，但好奇的玩家可以深挖
- **因果链本身就是奖励**：看到自己三步前埋下的伏笔终于引爆，这种因果快感是本作核心爽点

## 四、单季度核心循环 (Quarter Run Loop)

每个季度的目标：**在资源耗尽前，使目标小队失去继续推进任务的能力。**

### 4.1 注意力分层架构 (Attention Triage) — V4.0 核心新增

**V3.1 拥有 10+ 条并行决策轴，如果每回合全部要求玩家手动操作，认知负荷将超出上限。** V4.0 的核心设计原则是：**任何时刻，玩家需要主动做的事不超过 3 件；其余系统要么在后台自动运转，要么只在关键阈值被触碰时才弹出提醒。**

这就是"注意力分层架构"——不是砍掉系统，而是把系统按**需要玩家干预的频率**分为三层：

#### 前台决策 (Foreground) — 每回合必做，上限 3 个

这是玩家每回合的"手术操作台"，需要全神贯注：

| # | 决策 | 对应系统 | 玩家动作 |
|---|------|---------|---------|
| 1 | **选择情报投资** | 主动情报系统（雾灯模型） | 阅读免费摘要和热度信号 → 决定是否花 AP 解锁精确情报（私聊全文/深度扫描/元数据），剩余 AP 用于出牌 |
| 2 | **打出卡牌** | 卡牌系统 + 操作链 | 从手牌中选卡 → 拖向社交节点 → 系统自动判定方向（破坏/保护） |
| 3 | **选择破坏还是保护** | 双面立场系统 | 当拖向目标时，红/蓝光标自动呈现方向，玩家确认即可 |

**设计关键**：这三个决策覆盖了"看什么"、"做什么"、"为谁做"的完整决策闭环。玩家不需要在每回合关注其他任何事情。决策 #1 和 #2 共享同一个行动力池（详见 5.3 资源经济体系），"看多少"直接决定"能做多少"。

#### 告警决策 (Alert) — 后台运行，仅在阈值时弹出

这些系统在后台持续计算，用 UI 边缘指示器（如进度条、颜色变化）提供持续的被动感知。**只有当某个值接近危险阈值时**，系统才弹出一条简洁的告警，要求玩家在下一回合优先处理：

| 系统 | 后台显示方式 | 告警触发条件 | 弹出内容 |
|------|------------|------------|---------|
| **暴露 / 可疑度** | 屏幕角落的"监控眼"图标，颜色从绿→黄→红渐变 | 可疑度达到下一级 Awareness Level 前 3 点时 | ⚠️ "勇者队伍即将进入 Level 2 警觉——你的下一步操作可能被测试" |
| **魔王忠诚度** | 左下角魔王头像表情从微笑→平淡→皱眉→怒目 | 忠诚度降至下一阶段前 5 点时 | ⚠️ "魔王开始质疑你的效率——考虑在下次汇报中展示成果" |
| **MICE 培养进度** | 角色面板中的四维小雷达（只在切到情报视角时可见） | 某维度即将达到策反阈值时 | 💡 "该角色的 [自尊] 维度已接近策反窗口——这可能是你唯一的机会" |
| **资产维护** | 已策反资产的头像上显示一个小指示灯（绿→黄→红） | 资产满意度降至危险线时 | ⚠️ "你的资产 [盗贼] 正在动摇——她需要一次安抚操作，否则可能叛变" |
| **团队凝聚力** | 群聊区顶部的气氛条（从和谐蓝→紧张黄→危机红） | 凝聚力降至 30 以下时 | ⚠️ "队伍接近解散边缘——任何事故都可能成为最后一根稻草" |

**设计关键**：
- 告警是**建议而非强制**——玩家可以无视告警继续执行自己的计划，但后果自负
- 同一时间最多显示 **2 条告警**（优先级排序：即时危险 > 倒计时危险 > 机会窗口）
- 告警在回合开始时集中显示 1 秒后淡出到屏幕边缘，不打断操作流

#### 被动系统 (Passive) — 完全自动，零操作

这些系统在后台自动运行，玩家通过视觉反馈感知其状态，但**永远不需要手动操作**：

| 系统 | 运行方式 | 玩家感知方式 |
|------|---------|------------|
| **信任矩阵** | 角色间信任值根据社交节点自动增减 | 关系线的粗细/颜色自动变化 |
| **压力值** | 角色压力根据事件自动累积/衰减 | 角色头像下方的压力条自动变化 |
| **人格激活** | 人格弱点在条件满足时自动触发 | 角色头像闪烁对应人格图标 + 群聊中出现特征性发言 |
| **连锁反应 / 棋子化** | 被操纵角色自发产生冲突节点 | 群聊中自动出现新的社交节点（带 ⚡ 标记表示是连锁生成的） |
| **优雅度评价** | 根据玩家操作自动计算 | 回合结束时右下角简短闪过当前评级（粗暴/务实/精妙/大师） |

**设计关键**：被动系统是游戏"深度感"的来源——玩家不需要操作它们，但能从视觉反馈中**感受到自己的操作正在引发多米诺骨牌效应**。

#### 特殊处理：渗透者 & 汇报节点

两个 V3.1 的系统被重新定义，避免它们挤占前台决策空间：

| 系统 | V3.1 问题 | V4.0 处理 |
|------|----------|----------|
| **渗透者** | 每回合需手动下达 4 种指令（发言/私聊/传递情报/破坏） | 改为 **策略设定模式**（详见第六章）：玩家在渗透者入场时设定行为倾向，之后渗透者自动执行，仅在关键分叉弹出确认 |
| **汇报节点** | 定义模糊，像是回合内的额外行动 | 改为 **地图事件节点**（详见 8.3）：在季度小地图上作为固定遭遇出现，有自己的专属 UI 和决策空间，不占用正常回合 |

#### 认知负荷审计 (Cognitive Load Audit)

用此表验证任何一个回合中玩家的注意力分配是否健康：

| 层级 | 决策数量 | 认知消耗 | 状态 |
|------|---------|---------|------|
| 前台 | 3 | 高（需主动思考） | ✅ 在人类工作记忆极限（4±1）之内 |
| 告警 | 0-2 | 中（扫一眼即可） | ✅ 不强制行动，仅提供方向感 |
| 被动 | 5 | 零（自动运行） | ✅ 通过视觉反馈提供深度感，不消耗决策资源 |
| **合计** | **3 + 0~2 + 0** | — | ✅ 最坏情况 5 个注意力单元，远低于 V3.1 的 10+ |

### 4.2 单回合四阶段

V3.0 将 V2.1 的三阶段扩展为四阶段，V4.0 在此基础上标注了每个阶段涉及的**注意力层级**：

#### 阶段一：侦察部署 (Recon Phase) — 🔴 前台决策 #1

- 玩家决定本回合的**行动力**如何在情报与干预之间分配（详见 4.3 主动情报系统、5.3 资源经济体系）
- 决定监听哪些频道、对哪些角色进行深度扫描——每一点花在情报上的行动力，就是少一点用于出牌的行动力
- 可选择从**经费储备**中提取额外行动力（每回合最多 +2），以应对关键回合
- 🔔 此时告警系统集中显示上回合积累的告警（如暴露临界、忠诚度下降等）

#### 阶段二：情报监听 (Wiretapping Phase) — 被动接收

- 根据侦察部署的结果，系统展示对应频道的群聊内容
- 未被监听的频道只显示模糊摘要或"[加密通讯]"
- 系统从可见消息中提炼出社交节点（承诺、命令、抱怨、请求等）
- 暴露当前可见的压力值、信任波动与潜在爆点
- **关键差异**：你可能因为带宽不足而漏掉某些关键信息
- 📊 被动系统的视觉反馈在此阶段更新（关系线变化、压力条变化、人格图标闪烁）

#### 阶段三：HR 干预 (Intervention Phase) — 🔴 前台决策 #2 & #3

- 玩家消耗剩余行动力与手牌进行干预 → **前台决策 #2（打出卡牌）**
- 拖向社交节点时系统自动判定破坏/保护方向 → **前台决策 #3（确认立场方向）**
- 干预分为四类（详见第五章 卡牌系统）：
  - **信息操作卡**：撤回、伪造、延迟、泄露
  - **社交操作卡**：放大、栽赃、公开引用、舆论污染
  - **行政操作卡**：审计、人事调动、预算削减、绩效篡改
  - **外勤操作卡**：物资劫持、战场干预、装备破坏
- 每次操作会积累**暴露值**（告警系统后台追踪，详见 4.4 暴露风险系统）
- 支持**操作链 combo**（详见 4.6 操作链系统）

#### 阶段四：执行与复盘 (Execution & Replay Phase) — 被动观察

- 勇者小队执行本回合行动
- 右侧事故复盘终端展示结果速报 + 可选因果链
- 出现 Reaction Window 时允许少量即时干预
- 结算后进入**战后余波**：角色在群聊中对事故进行反应，生成下一回合的情绪状态
- 📊 被动系统在此阶段结算：信任变化、压力增减、人格激活、连锁反应、优雅度更新

### 4.3 主动情报系统 (Active Intel) — 雾灯模型 (Fog Lamp)

V2.1 中玩家默认看到所有群聊内容，这使得决策退化为优化题。V3.0 引入情报获取成本，V4.1 将其与卡牌操作整合到统一的**行动力**池中（详见 5.3 资源经济体系）。

#### 核心原则：免费概要 + 付费精度

**信息系统的目的不是隐藏内容，而是区分"直觉"和"精确"。** 所有频道永远可见，但有两层信息密度。AP 不决定"能不能看"，而是决定"看多细"：

| 频道类型 | 免费层（被动获取） | 付费层（消耗 AP） | AP 消耗 |
|----------|------------------|------------------|---------|
| **群聊** | 全文 + 自动提炼节点 | 同左（永远免费） | 0 |
| **私聊** | 一句话摘要 + 🔴🟡🟢 热度信号 | 全文 + 隐藏矛盾 + 可操纵节点 | 1 AP / 对 |
| **深度扫描** | 外在行为观察（"看起来压力很大"） | 内心独白 + 精确触发阈值 + 情报拼图碎片 | 2 AP |
| **元数据** | 大致印象（"盗贼和法师似乎在变近"） | 精确社交频率 + 趋势变化 + 孤立指标 | 1 AP |

**关键差异**：未付费的私聊不再显示 `[加密通讯 — 内容未知]`，而是显示有意义的摘要。玩家**永远知道发生了什么**，只是不知道具体细节。

#### 热度信号系统

免费摘要自带**热度指示器**，帮助玩家在投资前做有依据的判断，而不是盲赌：

| 热度 | 摘要示例 | 暗示 |
|------|---------|------|
| 🔴 高 | "战士 ↔ 牧师 私聊 \| **气氛激烈，有人提高了声量**" | 大概率有战术情报，值得投资 |
| 🟡 中 | "法师 ↔ 盗贼 私聊 \| **有些微妙，对话比平时长**" | 可能有战略或档案情报 |
| 🟢 低 | "战士 ↔ 法师 私聊 \| **简短交流，公事公办**" | 大概率只有档案情报 |
| 🔵 特殊 | "牧师独自沉默了很久 \| **深度扫描可能揭示内心变化**" | 明确暗示深扫有料 |

#### "空包弹"防御：三种情报货币

**设计铁律：任何花了 AP 的情报频道，必定返回至少一条此前不可见的有价值信息。** 但"价值"不只是"立刻能用"：

| 情报货币 | 定义 | 示例 | 何时兑现 |
|---------|------|------|---------|
| **战术情报** | 本回合立刻可操作 | "牧师的崩溃阈值还差 1 次甩锅" | 当场 |
| **战略情报** | 未来 2-3 回合可用 | "法师在试探盗贼结盟——裂痕种子" | 2-3 回合后 |
| **档案情报** | 跨季度生效的角色认知 | "牧师内心渴望被认可"（→ MICE E 型条件） | 本季度后期或下季度 |

即使角色"当前状态稳定"，深度扫描也必须返回档案情报——每次投资都会解锁角色情报拼图中的一块碎片（详见 9.4 角色情报拼图系统）。

**UI 重新框定"排除型"收获**：

- ❌ 错误表达："深度扫描完成。未发现明显弱点。"
- ✅ 正确表达："深度扫描完成。✓ 当前心理状态：稳定（无即时利用窗口）。✦ 新发现：她随身携带一封旧信 → 已录入情报拼图。💡 建议：将行动力优先投向其他目标。"

#### 行动力分配的策略价值

因为监听和出牌竞争同一个行动力池，每回合的分配方式本身就是核心策略。但现在有了雾灯的免费层，策略空间更加丰富：

- **重情报（2-3 AP 监听，0-1 AP 出牌）**：从摘要升级到全文，获得精确操作条件——适合"蓄力回合"，将模糊的直觉转化为精准的弹药
- **重干预（0 AP 监听，3 AP 出牌）**：依靠免费摘要和热度信号做粗略判断，全力输出——适合"收割回合"，信息足够多时不需要更精确
- **精确狙击（1 AP 解锁一个 🔴 热度私聊 + 2 AP 出牌）**：只投资最有料的频道——适合大部分普通回合
- **爆发（从经费储备提取 +2 AP → 本回合 5 AP）**：同时深度扫描 + 打出双卡 combo——但消耗季度储备，不可持续

**雾灯模型的核心价值**：新手不付费也能基于摘要做出"还行"的决策（粗暴路线）；高手通过精准投资获得精确情报，做出"精妙"和"杰作"级操作。**AP 买到的不是"能不能做"，而是"做得多漂亮"。**

### 4.4 暴露风险系统 (Exposure Risk)

V2.1 中 Admin 操作无代价。V3.0 引入暴露系统，让每次操作都成为风险-收益权衡：

#### 可疑度 (Suspicion)

- 每次使用管理员权限或卡牌操作，会积累**可疑度**（0-3 点/次）
- 可疑度在季度内持续累积，不会自动衰减
- 不同操作的可疑度成本不同：

| 操作类型 | 可疑度 | 说明 |
|----------|--------|------|
| 延迟消息 | +0 | 网络延迟很常见，几乎不会被怀疑 |
| 已读不回 | +0 | 无法追溯到外部干预 |
| 放大/公开引用 | +1 | 略显刻意，但可解释为系统推送 |
| 撤回消息 | +1 | 发送者可能察觉"我没撤回啊？" |
| 伪造消息 | +2 | 高风险，内容风格不匹配可能被识破 |
| 禁言 | +2 | 明显的外部干预 |
| 篡改绩效/审计 | +3 | 行政操作留有痕迹 |

#### 勇者觉察等级 (Awareness Level)

当可疑度达到阈值时，勇者队伍整体进入更高的觉察等级：

- **Level 0（默认）**：勇者对通讯安全无感知，所有操作正常生效
- **Level 1（可疑度 ≥ 10）**：
  - 多疑型角色开始使用私聊代替群聊讨论敏感话题
  - 部分群聊消息变得含糊（"你知道的那件事"代替具体计划）
  - 玩家对公开群聊的信息获取量下降约 30%
- **Level 2（可疑度 ≥ 20）**：
  - 队伍中可能出现"反间意识"——角色开始测试通讯是否被窃听（发送诱饵消息）
  - 如果玩家对诱饵消息做出反应（撤回/伪造），可疑度额外 +5
  - 伪造消息的成功率下降（角色会互相确认）
- **Level 3（可疑度 ≥ 30）**：
  - 队伍进入"通讯管制"：关键战术讨论转为线下面谈，群聊中只剩日常闲聊
  - 玩家的信息操作卡效力减半
  - 但同时队伍因为"知道被监视"而产生额外压力（+压力给全队）
  - **如果玩家安插了渗透者**，渗透者面临被排查的风险

#### 暴露系统的策略意义

- 创造了**激进 vs 保守**的核心张力：激进操作推进快但暴露高，保守操作安全但可能被勇者自我修复
- 鼓励"优雅"的操作方式：低暴露的精妙 combo 比高暴露的粗暴操作更有价值
- 不同季度的暴露预算不同：Q1 阈值高（勇者新手不警觉），Q4 阈值低（精英队伍有反间意识）

### 4.5 左侧群聊：主战场 (Social Combat Layer)

左侧通讯群聊不是叙事包装，而是本作真正的**主战场**。玩家进行的是一场围绕承诺、信任、压力、权威和信息传递的**社交工程战**。

#### 核心博弈对象

- **承诺**：社交契约（"我给你加盾""你先扛我第二拍到"），是右侧事故的主要因果来源
- **信任**：决定角色是否相信队友、是否愿意照计划行动
- **压力**：压力越高，角色越容易误读、失控、崩溃或进入可招安状态
- **权威**：话语权等级差异，影响命令效力与背书权重
- **注意力 / 信息传递**：谁看到了什么、谁的消息被淹没、哪句话被置顶、哪条私聊被泄露

#### 社交节点

群聊的基础战斗单位是**可操作的社交节点**：

- 一个治疗承诺
- 一个冲锋命令
- 一次资源分配
- 一条抱怨
- 一次求救
- 一条私聊泄露

玩家干预的对象是这些节点本身，节点被操纵后反向改变人物关系与执行结果。

#### 中间胜态

左侧群聊有自己的"战果"层级，从轻到重：

| 胜态 | 含义 | 效果 |
|------|------|------|
| **违约** | 关键承诺未兑现 | 事故概率上升，信任开始受损 |
| **错位** | 角色行动节奏不再统一 | 战斗效率下降，压力上升 |
| **失声** | 重要信息未能传达给关键成员 | 角色做出错误决策 |
| **甩锅** | 事故责任集中到某人身上 | 被甩锅者压力飙升，其他成员信任下降 |
| **孤立** | 角色被团队边缘化 | 孤立者战斗意志下降，进入可招安前置 |
| **派系** | 队伍分裂为两个对立阵营 | 凝聚力加速下降，内部命令可能被抵制 |
| **动摇** | 角色开始怀疑是否还应继续 | 接近可招安状态 |
| **可招安** | 角色不再把魔王阵营视为绝对敌人 | 可使用招安卡牌 |

#### 勇者方的反制

勇者队伍具备自我修复能力，形成真正的博弈：

- **澄清**：解释误会、否认伪造、恢复消息真实性
- **背书**：队友作证修复被破坏的信任
- **安抚**：压下争吵、降低压力、阻止内讧升级
- **重新分工**：计划被打乱后快速重建协作秩序
- **团建**（V3.0 新增）：在休整点进行主动修复，恢复凝聚力
- **反间排查**（V3.0 新增）：在 Awareness Level 2+ 时主动调查通讯异常

玩家的核心乐趣：**抢在勇者完成修复前，把矛盾推过临界点。**

#### 连锁反应与棋子化 (Cascade & Pawn)

**V3.0 新增核心机制。**

当一个角色因为你的操纵而产生强烈负面情绪时，该角色可能自发地在群聊中产生新的冲突节点：

- 被甩锅的战士可能主动在群聊中指责法师 → 生成新的 `抱怨` 节点
- 压力过高的牧师可能拒绝下一次治疗承诺 → 生成新的 `拒绝` 节点
- 被孤立的盗贼可能公开要求重新分配战利品 → 生成新的 `资源冲突` 节点

**这些角色生成的节点不会增加你的可疑度**（因为它们是角色自发行为），但你可以对这些新节点继续施加操作。

这就形成了本作的终极策略循环：

```
你的操作 → 角色情绪变化 → 角色自发产生冲突 → 你在新冲突上继续操作 → 更大的连锁反应
```

**棋子化**是连锁反应的高级形态：当一个角色的特定情绪（愤怒/猜疑/贪婪）持续处于高位时，该角色会变成半自动的"棋子"——每回合固定产生 1-2 个与其情绪对应的冲突节点，无需你额外操作。

**棋子化的策略意义**：
- 减少你每回合需要亲自操作的次数 → 节省行动力和暴露
- 使冲突看起来完全"有机" → 不增加可疑度
- 但棋子化角色可能失控（产生你不想要的冲突，或自行离队）

#### 优雅度评价 (Elegance Rating)

**V3.0 新增核心机制。**

每回合结算时，系统会评估玩家操作的"优雅度"——衡量的不是你造成了多少伤害，而是你的操纵有多精妙：

| 等级 | 定义 | 条件示例 | 奖励 |
|------|------|----------|------|
| **粗暴 (Brute)** | 直接使用高暴露操作达成目标 | 伪造命令 + 禁言反对者 | 目标达成，但暴露 +4-6 |
| **务实 (Pragmatic)** | 使用中等暴露操作配合基础 combo | 撤回承诺 + 放大抱怨 | 目标达成，暴露 +2-3 |
| **精妙 (Subtle)** | 使用低暴露操作通过多步链条达成目标 | 延迟消息 + 利用角色自发冲突 + 在余波中顺势推进 | 目标达成，暴露 +0-1，额外卡牌奖励 |
| **杰作 (Masterwork)** | 仅用 1-2 次操作触发连锁反应，角色自行走向崩溃 | 一次精准泄露 → 角色棋子化 → 角色自行摧毁团队 | 目标达成，暴露 +0，解锁稀有卡牌 |

**优雅度的设计意义**：
- 让新手和高手面对同一局面时有截然不同的解法
- 为"重打同一季度"提供挑战价值——这次能用更优雅的方式通关吗？
- 与暴露系统联动：优雅操作的低暴露为后续回合保留了更多操作空间

#### 群聊信息的三层结构

- **第一层：聊天流** — 角色魅力、代入感、戏剧性
- **第二层：提炼后的社交节点** — 一眼可读的策略信息
- **第三层：关系 / 状态板** — 长期变量（信任、压力、权威、创伤、可招安性）

设计原则：**文本负责好看，结构化信息负责能玩。**

### 4.6 操作链系统 (Operation Chain)

**V3.0 核心新增系统。**

V2.1 的 5 种操作各自独立，缺乏组合深度。V3.0 引入**三拍节奏**的操作链：

#### 三拍节奏：布置 → 触发 → 放大

| 节奏层 | 说明 | 卡牌特征 |
|--------|------|----------|
| **布置 (Setup)** | 改变某个节点的状态，创造后续操作的条件 | 低费用、低暴露、单独效果弱 |
| **触发 (Trigger)** | 利用已布置的条件，引爆一次有意义的事故 | 中费用、中暴露、需要特定前置 |
| **放大 (Amplify)** | 在事故发生后扩大影响，将个人事故转化为团队危机 | 高费用、高暴露、效果翻倍但需要时机 |

#### 操作链示例

**链条 1：信任崩塌链**
1. [布置] `延迟` 牧师的治疗承诺（牧师的承诺比战士的冲锋命令晚到 → 战士不知道牧师会支援）
2. [触发] 执行阶段：战士冲锋后缺少治疗 → 触发事故 → 战士对牧师产生不信任
3. [放大] 战后余波中，对战士的抱怨使用 `公开引用` → 不信任从私人矛盾升级为公开对质

**链条 2：身份伪装链**
1. [布置] `泄露` 法师的一条私聊（法师曾私下抱怨队长指挥能力差）
2. [触发] `伪造` 一条"队长看到了泄露内容"的系统通知 → 法师以为自己已经被队长知晓
3. [放大] 法师因恐惧主动在群聊中辩解 → 反而让更多人知道了这件事（角色自发产生的连锁节点）

**链条 3：替罪羊链**
1. [布置] `审计` 盗贼的履历（暴露其过往黑历史）
2. [触发] 当战斗出现任何事故时，使用 `栽赃` 将责任引向盗贼
3. [放大] 队伍其他成员看到审计结果后自然倾向于相信"盗贼果然不可靠"→ 盗贼被孤立

#### 操作链的设计价值

- **两张牌 > 两张牌之和**：combo 产生的效果显著大于单独使用
- **鼓励提前规划**：布置阶段的卡牌单独打效果很弱，但如果你能在下回合触发它，价值翻倍
- **创造"忍住不出"的 tempo 决策**：手里有一张触发卡但前置条件还没布置好——是现在打出一个弱效果，还是攒到下回合打一套完整 combo？
- **构筑方向**：玩家可以围绕某类操作链构建牌组（信任崩塌流、替罪羊流、信息战流）

### 4.7 Reaction Window（反应窗口）

执行阶段允许玩家继续干预，但必须通过**事件窗口**触发。

#### 触发原则

- 只在关键节点出现
- 每次提供 2-3 个直观的反应选择
- 目的是放大戏剧性和补刀，而不是重做一遍主策略判断

#### 典型触发时机

- 战士即将冲锋 / 牧师准备治疗
- 某成员血量低于阈值
- 两名成员的信任跌破阈值
- 角色进入可招安状态
- 计划阶段埋下的"布置"条件被满足

#### 反应窗口中的干预

- **环境卡**：增援呼叫、地板塌陷、封门、照明故障
- **反应卡**：断线、误报、恐吓广播、人事冻结
- **条件触发卡**：招安话术、火上浇油、补刀舆论

#### 设计禁区

- 不应要求玩家阅读大量文本
- 不应让玩家重新做完整策略规划
- 不应把执行阶段扩张成第二套卡牌主循环

### 4.8 胜利条件 (Victory Conditions)

| 条件 | 描述 | 评价 | 战役影响 |
|------|------|------|---------|
| **团灭 (Wipeout)** | 全员 HP 归零 | 最低评价 | 角色永久死亡，后续季度无法再出现 |
| **解散 (Disband)** | 团队凝聚力归零 | 较高评价 | 角色存活但关系破裂，可能在后续季度以敌对/中立身份出现 |
| **招安 (Recruitment)** | 关键成员被策反至魔王阵营 | 最高评价 | 招安角色在后续季度提供情报/卡牌/权限 |

新增评价维度：
- **优雅度总分**：整个季度的操作优雅度累计，影响奖励卡牌品质
- **暴露控制**：季度结束时的可疑度越低，下一季度的初始带宽越高

### 5.3 资源经济体系 (Resource Economy) — V4.1 核心重构

**V4.0 的"预算"和"情报带宽"是两个平行但独立的回合资源，缺乏经营感。** V4.1 将资源重构为三层架构：回合级（战术决策）、季度级（中期经营）、跨季度级（战略积累），让玩家像真正的情报主管一样管理有限的组织资源。

#### 第一层：回合级资源 — 每回合的战术决策

| 资源 | 获取 | 规则 | 设计意图 |
|------|------|------|----------|
| **行动力 (Action Points)** | 每回合固定 3 点 | 不结转；用于监听频道 AND 打出卡牌 | 统一"看什么"和"做什么"的资源池，迫使"侦察 vs 干预"的核心取舍 |
| **手牌 (Hand)** | 每回合抽 3 张 | 手牌上限 7 张，未使用的保留 | 手牌是持续性资产，行动力是消耗性预算 |

**关键变化**：V4.0 的"预算 3 点 + 带宽 3 点"合并为"行动力 3 点"。监听私聊要花 1 AP，深度扫描要花 2 AP，打出一张 1 费卡也要花 1 AP——它们竞争同一个资源池。

**"看 vs 做"取舍**示例（雾灯模型下）：

- 你有 3 AP。不监听任何频道（0 AP）→ 依靠免费摘要和热度信号做粗略判断，全部 AP 用于出牌——信息模糊但干预强
- 你有 3 AP。解锁 1 个 🔴 热度私聊（1 AP）+ 2 AP 出牌——精确狙击最有料的频道
- 你有 3 AP。全部投入情报（深扫 2 + 元数据 1）→ 0 AP 出牌，只能用 0 费卡——本回合是蓄力回合
- 你有 3 AP + 经费储备提取 2 AP → 本回合 5 AP——同时获得精确情报和打出 combo，但消耗季度储备

#### 第二层：季度级资源 — 中期经营与爆发窗口

| 资源 | 获取 | 消耗方式 | 设计意图 |
|------|------|---------|----------|
| **经费储备 (Reserve Fund)** | 每季度初始获得总量（Q1: 8, Q2: 10, Q3: 12, Q4: 15） | 主动"提取"到行动力（1:1 兑换），每回合最多提取 2 点 | 季度总量制造"细水长流 vs 关键回合爆发"的经营感 |
| **暗线网络 (Shadow Network)** | 成功的低暴露操作积累（精妙评价 +2，杰作评价 +4） | 消耗暗线点数发动"暗线操作"——效果与普通操作相同，但暴露 = 0 | 奖励优雅玩法的"政治资本"，用积累换安全 |
| **权限等级 (Clearance)** | 按季度自动解锁 + 特殊成就奖励 | 不消耗；解锁更多卡牌和情报频道的使用权 | 非消耗性进阶，控制系统复杂度的渐进释放 |

**经费储备的经营逻辑**：

- 季度初你有 10 点储备。如果每回合都提取 2 点，储备在第 5 回合耗尽——后续回合只有基础 3 AP
- 如果前 3 回合只用基础 AP 忍耐，储备完好——第 4 回合可以爆发 5 AP，打出一套完整 combo
- 如果整个季度都没用储备——季度结算时剩余储备转化为额外优雅度评价

**暗线网络的经营逻辑**：

- 暗线点数跨回合保留（但不跨季度）
- 每次暗线操作消耗对应操作原本暴露值的 2 倍暗线点数
- 例：伪造（原暴露 +2）用暗线执行 = 消耗 4 暗线点，暴露 +0
- 暗线网络 = 用"过去的优雅"购买"未来的安全"

#### 第三层：跨季度资源 — 战役级积累

| 资源 | 来源 | 效果 | 设计意图 |
|------|------|------|----------|
| **人事档案 (Dossier)** | 深度扫描、策反、长期监听积累的情报拼图碎片（详见 9.4） | 已知角色再次出现时，情报消耗降低；部分拼图碎片直接可见 | 情报资产的长期回报 |
| **魔王信任 (Loyalty)** | 每季度初始 100，破坏操作维持/回升，保护操作下降 | 影响下季度初始经费储备（信任高→储备多）和起始权限 | 与双面立场系统联动，见第八章 |
| **公会档案 (Guild Intel)** | 公会根据玩家历史行为自动积累 | 影响下季度敌方初始觉察等级和反制能力 | 与反情报升级系统联动，见 11.5 |

**跨季度联动**示例：

- Q1 暴露控制得好（公会档案低）→ Q2 初始觉察 Level 0，起步轻松
- Q1 魔王信任高（忠诚线）→ Q2 经费储备 +3，更多操作空间
- Q1 招安成功（人事档案丰富）→ Q2 如果遇到同一角色，深度扫描免费

#### 资源流动全景

```
季度初始
  ├─ 经费储备 [总量] ← 受上季度魔王信任影响
  ├─ 权限等级 [解锁内容] ← 季度自动 + 成就奖励
  └─ 人事档案 [信息] ← 上季度积累

每回合
  ├─ 行动力 3 点 ← 固定生成（不结转）
  ├─ + 经费提取 0-2 点 ← 从季度储备中提取
  ├─ 手牌抽 3 张 ← 上限 7 张（结转）
  └─ 暗线网络 [积累] ← 低暴露操作奖励
        ↓
  行动力 → 花在监听 OR 出牌 → 产出情报 / 制造事故
        ↓
  暗线操作 → 花暗线点 → 零暴露执行操作

季度结算
  ├─ 魔王信任 → 影响下季度经费储备
  ├─ 公会档案 → 影响下季度敌方反制
  └─ 人事档案 → 影响下季度情报成本
```

#### 卡牌费用与行动力

V4.0 的卡牌"费用"现在直接消耗行动力：

- 费用 0 的卡（延迟、已读不回）= 消耗 0 AP → 与监听不竞争，是"永远买得起的底牌"
- 费用 1 的卡（撤回、放大、窃听升级）= 消耗 1 AP → 和监听一个私聊同价
- 费用 2 的卡（伪造、审计、栽赃）= 消耗 2 AP → 和深度扫描同价

**一次深度扫描（2 AP）和一次伪造（2 AP）在行动力上是等价的。** 你不能同时做这两件事（除非提取经费储备），这创造了"情报 vs 操作"的永恒张力。

#### 手牌

- 每回合从牌库抽取手牌（初始抽 3 张）
- 未使用的手牌保留到下回合（但手牌上限为 7 张）
- 地图休整点可进行删牌/升级牌/获取新卡

#### 卡牌升级

部分卡牌可升级为更强版本：

- `撤回` → `无痕撤回`（暴露从 +1 降到 +0）
- `伪造` → `深度伪造`（可模拟目标角色的语言风格，降低被识破概率）
- `挑拨` → `离间计`（同时影响双方，而不只是单方面误解）
- `栽赃` → `完美栽赃`（自动附带伪造证据，无需额外 combo）

### 5.4 构筑流派

卡组构筑的方向不是"取最强卡"，而是围绕特定打法路线选择 combo 配件：

| 流派 | 核心思路 | 关键卡牌 | 优势 | 劣势 |
|------|---------|---------|------|------|
| **信息战 (Disinfo)** | 通过伪造和延迟制造认知混乱 | 伪造、延迟、撤回、窃听升级 | 暴露低、连锁性强 | 需要精确时序，门槛高 |
| **替罪羊 (Scapegoat)** | 集中火力孤立一人 | 审计、栽赃、人事背锅、放大 | 快速推进到"孤立"胜态 | 如果目标被队友保护则失效 |
| **压力战 (Burnout)** | 全队均匀施压直到有人崩溃 | 已读不回、预算削减、加班令、增援呼叫 | 不依赖特定裂痕 | 进度慢、给勇者修复时间 |
| **招安大师 (Recruiter)** | 精准培养一个角色的可招安条件 | 泄露、挑拨、招安话术、深度扫描 | 最高评价路线 | 需要长线规划，中途被干扰则前功尽弃 |
| **棋子流 (Puppeteer)** | 快速棋子化一个角色，让他替你制造混乱 | 放大、栽赃、人格触发卡 | 后期零暴露高产出 | 棋子化角色可能失控 |
| **守护者 (Guardian)** | 暗中保护某个角色，同时向魔王交出"成绩单" | 反向撤回、反向伪造、已读不回（压制挑衅）、渗透者关怀 | 开启共情线/双面间谍结局 | 魔王忠诚度下降，需要牺牲其他角色维持掩护 |

### 5.5 卡牌双面性 (Dual-Use Cards)

**V3.1 核心新增机制。**

本作的每一张卡牌都不只有"破坏"一种用法。当玩家选择打出一张卡时，系统会根据**目标节点的性质**自动呈现两种操作方向：

| 卡牌 | 破坏用法（对敌） | 保护用法（对友） |
|------|----------------|----------------|
| **撤回** | 撤回治疗承诺 → 制造事故 | 撤回队友对目标的恶言 → 阻止冲突升级 |
| **伪造** | 伪造假命令 → 制造混乱 | 伪造队长的道歉 → 修复裂痕 |
| **延迟** | 延迟支援信息 → 制造事故 | 延迟刻薄的话 → 让怒气冷却后再传达 |
| **泄露** | 泄露私聊 → 制造猜疑 | 泄露某人私下说的好话 → 修复信任 |
| **已读不回** | 让求救无人回应 | 让挑衅无人搭理 → 阻止争吵升级 |
| **放大** | 放大抱怨 → 扩散矛盾 | 放大某人的感谢/认可 → 提振士气 |
| **审计** | 揭露黑历史 → 制造把柄 | 审计发现贡献被低估 → 恢复权威 |
| **公开引用** | 断章取义制造冲突 | 引用某人的善意发言 → 修复形象 |

#### 设计原则

- **不需要两套卡牌**：同一张卡、同一套费用和暴露值，只是应用方向不同
- **保护操作同样消耗行动力和暴露**：保护不是"免费的善良"，而是有机会成本的策略选择
- **保护操作不产生新的可疑度类型**：对勇者来说，无论你是在破坏还是保护，后台操纵本身就是可疑的
- **但保护操作会降低魔王忠诚度**（详见第八章双面立场系统）：你不是免费做好人，而是在赌自己的命

#### UI 表达

当玩家选择一张卡并拖向一个社交节点时，系统自动判断该操作属于"破坏"还是"保护"，并用颜色区分：

- 红色光标 → 破坏方向（对该角色/关系有害）
- 蓝色光标 → 保护方向（对该角色/关系有利）

玩家不需要切换模式，只需要选择"对谁用、用在哪个节点上"，系统根据语境自动判断方向。

## 六、渗透者系统 (Infiltrator System)

**Q2 季度解锁。V4.0 重构为"策略设定模式"，不再挤占每回合的前台决策空间。**

### 6.1 概念

除了外部的通讯管理员身份，玩家可以在勇者队伍中安插**渗透者**——一个伪装成勇者或随行人员的魔王军特工。

渗透者不是第二个玩家操控的战斗单位，而是一个**按策略自动执行的深层代理人**。

### 6.2 策略设定模式 (V4.0 重构)

V3.1 中渗透者每回合需手动下达发言/私聊/传递情报/破坏四种指令，这使得它实质上成为第四个前台决策。V4.0 将其改为**设定一次、自动执行、关键时刻弹出确认**的策略模式：

#### 入场设定

当渗透者首次安插进队伍时，玩家进行一次性的策略配置：

| 维度 | 选项 | 效果 |
|------|------|------|
| **社交倾向** | 亲和型 / 煽动型 / 沉默型 | 决定渗透者在群聊中的自动发言风格 |
| **目标锁定** | 指定 1 名主要亲近目标 | 渗透者会优先与该角色建立私聊关系 |
| **情报策略** | 被动监听 / 主动探查 | 被动 = 免费监听 1 个额外频道；主动 = 尝试套话获取深度情报（暴露更高） |
| **激进度** | 保守 (1-3) / 标准 (4-6) / 激进 (7-10) | 控制渗透者自动执行破坏行为的频率和力度 |

#### 自动执行

设定完成后，渗透者每回合自动执行行为，表现为群聊中的自然发言和行动：

- **亲和型**：安慰受伤成员、附和正面发言、建立信任 → 暴露低、为 MICE E 型策反铺路
- **煽动型**：附和抱怨、质疑决策、制造"合理的"不满 → 暴露中、加速凝聚力下降
- **沉默型**：最少发言、只在关键时刻表态 → 暴露极低、影响力也低

#### 关键分叉弹出

渗透者在以下情况自动暂停并弹出确认窗口，**将决策权交还玩家**：

| 触发条件 | 弹出内容 | 示例 |
|----------|---------|------|
| 遇到高价值情报 | "渗透者发现 [重要信息]——要传回吗？（+暴露 2）" | 发现队长在私下联络公会总部 |
| 可执行关键破坏 | "渗透者有机会 [破坏行动]——执行吗？（+暴露 3）" | 可以在战斗中"失误"导致牧师受伤 |
| 暴露值临近阈值 | "渗透者暴露度已达 70%——要降低激进度吗？" | 多疑型角色开始频繁观察渗透者 |
| 目标角色寻求帮助 | "你的亲近目标向渗透者求助——如何回应？" | 被保护的角色向渗透者倾诉压力 |

**设计效果**：渗透者从"每回合多 4 个按钮"变成了"设定一次 + 偶尔弹出 1 个关键选择"，认知负荷从 **每回合 4 次操作** 降至 **每 2-3 回合 1 次确认**。

#### 策略调整

玩家可以在任何回合的侦察部署阶段（阶段一）**重新配置渗透者策略**，但每次调整有 1 回合的执行延迟（渗透者需要"自然过渡"到新的行为模式）。频繁调整本身也会微增暴露值。

### 6.3 渗透者的能力（自动执行清单）

渗透者的四种能力在策略设定模式下自动运行：

- **发言**：根据社交倾向设定，在群聊中以自然身份发言（频率和风格由激进度和倾向决定）
- **私聊**：与目标锁定的角色自动建立和维持私聊关系
- **传递情报**：根据情报策略设定，无需消耗带宽即可监听一个额外频道
- **破坏**：根据激进度设定，在执行阶段以一定概率故意失误

### 6.4 渗透者的风险

- 渗透者的自动行为会积累自身的**暴露值**（激进度越高，暴露越快）
- 当暴露值达到阈值时，多疑型角色可能开始怀疑渗透者
- 一旦渗透者被识破：
  - 渗透者被驱逐
  - 全队可疑度直接飙升到 Level 3
  - 全队凝聚力因"共同敌人"而**回升**（最坏情况）
  - 本季度不可再安插新渗透者

### 6.5 渗透者的策略价值

- 渗透者的发言不增加通讯系统的可疑度（因为是"队内人员"的正常发言）
- 渗透者是"优雅度"最高的操作方式——通过角色扮演式的社交操纵替代暴力的 Admin 权限
- V4.0 策略设定模式让渗透者从"负担"变成了"投资"——前期花一次精力设定，后续每回合自动产出价值

### 6.6 渗透者与棋子化的区别

| | 渗透者 | 棋子化角色 |
|---|---|---|
| 身份 | 你的特工 | 被操纵的勇者 |
| 控制方式 | 策略设定 + 关键分叉确认（V4.0） | 半自动（角色自发但可引导） |
| 暴露风险 | 可能被识破并反噬 | 无暴露（是角色自发行为） |
| 持续性 | 直到被驱逐 | 直到角色情绪稳定 |
| 策略定位 | 长线投资型代理人 | 自走式炸弹 |
| 注意力层级 | 告警层（仅关键分叉弹出） | 被动层（完全自动） |

## 七、情报官操作体系 — MICE 策反模型 (Intelligence Operations)

**V3.1 核心新增系统。** 取代 V3.0 的简化招安机制。

策反（招安）不是"推到阈值打出一张卡"的一次性事件，而是一个跨越多个回合的**情报经营过程**。本章引入真实情报学中的 MICE 四维策反模型和资产发展周期，让策反路线成为与破坏路线同等深度的完整策略线。

### 7.1 MICE 四维模型

MICE 是现实情报机构用于评估和策反目标的经典框架：

| 维度 | 含义 | 适用人格 | 策反后的资产质量 |
|------|------|---------|----------------|
| **M — Money（金钱/利益）** | 用物质利益诱惑 | 贪婪、实用主义 | 雇佣兵型：听话但忠诚度低，持续需要利益投入，断供可能反水 |
| **I — Ideology（理念）** | 让目标相信你的事业是对的 | 理想主义（幻灭后）、反叛 | 信仰型：忠诚度最高，但若发现魔王军也不是"好人"会崩溃 |
| **C — Coercion（胁迫/把柄）** | 用威胁、黑材料或无退路逼迫 | 有黑历史者、被孤立者 | 被迫型：执行力高但充满怨恨，长期有反噬风险 |
| **E — Ego（自尊/认同）** | 满足被忽视者的自尊需求 | 自卑、忠诚（被背叛后） | 依赖型：情感忠诚，但需要持续提供认同感，否则陷入新的自我怀疑 |

#### 与 V3.0 的对比

- V3.0 有三条招安路径（利益/情感/理念），V3.1 扩展为四维并加入**胁迫 (C)** 作为全新的、道德灰度最高的策反路线
- V3.0 的"情感型招安"在 V3.1 中被精确化为 E（自尊/认同），含义更清晰
- 四条路线产生**质量完全不同的资产**，直接影响后续季度的风险和回报

### 7.2 角色面板：双视角设计

**设计要点**：MICE 雷达图**不是**角色面板的默认信息。

每个角色面板有两个可切换视角：

| 视角 | 默认? | 展示内容 | 传达的信号 |
|------|:-----:|---------|-----------|
| **档案视角** | **是** | 名字、职业、性格、过往经历、当前情绪、与队友的关系叙述 | "这是一个有故事的人" |
| **情报视角** | 否 | MICE 四维雷达图、可攻击度百分比、发展进度条、弱点分析 | "这是一个可以被策反的目标" |

**为什么默认展示档案视角？**

如果系统默认展示"如何击溃这个人"的雷达图，玩家会把每个角色当成待破解的谜题而不是活生生的人。这会从根源上阻碍共情的产生，让共情线和保护线变成"只有对话选项不同的分支"而不是"由玩家内心驱动的真实选择"。

玩家主动切换到情报视角本身就是一个微型的道德选择："我要把这个人当工具来分析了。"

#### MICE 雷达图示例

深度扫描一个角色后，在情报视角中可见：

```
牧师的 MICE 雷达：
  M（利益）：15%  — 不太在乎钱，冒险报酬对她来说只是生存手段
  I（理念）：30%  — 对冒险体系有些动摇，但"拯救世界"的信念还在
  C（胁迫）：60%  — 有多次被甩锅的记录，可以构造"履历污点"作为筹码
  E（自尊）：85%  — 长期被忽视和否定，极度渴望被认可
```

一眼可知：E 是最大突破口，C 是备选路径。

### 7.3 资产发展周期 (Asset Development Cycle)

策反一个角色遵循情报学的标准五阶段：

```
Spot（发现） → Assess（评估） → Develop（培养） → Recruit（策反） → Handle（运营）
```

#### 阶段一：Spot（发现）

- 在情报监听阶段，当某个角色的特定 MICE 维度超过初始阈值时，角色头像上出现一个微小的裂缝图标
- 提示"此人可能可以被发展"
- 不需要新系统，纯 UI 提示

#### 阶段二：Assess（评估）

- 玩家通过深度扫描（消耗 2 点带宽）获取角色的 MICE 雷达图
- 判断哪个维度最容易突破
- 这一步的核心价值：**避免盲目投入**——如果你一直在用 E（自尊）路线培养一个实际上 M（利益）才是突破口的角色，进度会极慢

#### 阶段三：Develop（培养）

**这是策反路线的核心玩法阶段。**

每个可策反角色有 MICE 四维的**发展进度条**。玩家的操作会推进对应维度的进度：

| 操作 | 推进的 MICE 维度 | 效果 |
|------|-----------------|------|
| 泄露私聊（揭露队长对目标的差评） | E +15 | 自尊受损，渴望被认可 |
| 伪造公会通知"你的绩效排名末位" | E +10, C +10 | 自尊打击 + 制造焦虑 |
| 渗透者私下关心目标 | E +20 | "我觉得你做得很好，是他们不懂珍惜" |
| 审计揭露目标过往失误 | C +15 | 搜集日后可用于胁迫的材料 |
| 让目标目睹公会/王国的黑暗面 | I +20 | 理念动摇 |
| 通过"错发补给"让目标尝到甜头 | M +15 | 利益诱惑 |
| 让目标在战斗中被队友抛弃 | E +10, I +10 | "这个组织不值得我效忠" |
| 栽赃构造目标的"罪证" | C +20 | 积累可用于胁迫的把柄 |

当某一维度的进度条填满（100%）时，该维度的策反路径解锁。

**关键设计**：你可以同时推进多个维度，策反时混合使用（例如 E + M："你值得更好的，而且我们给得起"），产生更稳定但更昂贵的复合型资产。

#### 阶段四：Recruit（策反）

当至少一个 MICE 维度的进度条满格时，系统解锁**策反对话事件**。

这不是"打出一张招安话术卡"那么简单——它是一次小型的情报接头场景，玩家需要根据培养的维度选择正确的"开场白"（Pitch）：

| Pitch 类型 | 对应 MICE | 对话风格 |
|-----------|-----------|---------|
| **利益提案** | M | "魔王军可以给你真正的回报。不用再看别人脸色。" |
| **理念共鸣** | I | "你觉得公会真的在做对的事吗？也许换一个角度，你才能真正实现自己的理想。" |
| **胁迫施压** | C | "你的这些记录……如果公会看到了，你觉得会怎样？但这不一定要发生。" |
| **自尊认同** | E | "你值得被看见。在那边没有人珍惜你，但我们知道你的价值。" |

使用与目标 MICE 最高维度匹配的 Pitch → 策反成功率最高  
使用不匹配的 Pitch → 成功率大幅下降，甚至可能触发反招安

#### 阶段五：Handle（运营）

**策反成功后，角色不立刻离队，而是成为你的"在编资产"(Active Asset)，继续留在队伍中。**

这是 MICE 体系带来的最大新增玩法空间——策反只是开始，运营才是持续的博弈。

**资产指令**：每回合你可以给资产下达一个轻量指令（从 2-3 个选项中选 1 个）：

- 传递错误信息给队友
- 在战斗中故意失误
- 搜集特定情报（免费获得一次深度扫描效果）
- 暗中保护另一名角色（如果你同时在走保护线）
- 不下指令（资产自行行动，风险最低）

**资产维护**：不同 MICE 类型的资产需要不同的维护方式：

| 资产类型 | 维护需求 | 不维护的后果 |
|---------|---------|------------|
| M 型（利益） | 每 2 回合消耗 1 行动力 | 断供后忠诚度快速下降，可能反水 |
| I 型（理念） | 你的行动不能违背其信仰 | 若你做了"太残忍"的事（如逼死无辜角色），I 型资产可能反水 |
| C 型（胁迫） | 无需主动维护 | 但每回合有固定概率反噬（向队友坦白、故意暴露你） |
| E 型（自尊） | 渗透者或私聊持续提供认同 | 若长期被忽视，陷入新一轮自我怀疑，可能主动暴露 |

**资产暴露风险**：资产在队内的异常行为可能被多疑型队友察觉。如果资产被识破：

- 资产被清除（驱逐或处决）
- 全队凝聚力因"共同敌人"回升
- 你的可疑度飙升

你需要在"让资产干更多活"和"保护资产不被发现"之间权衡。

### 7.4 反策反 (Counter-Recruitment)

某些高忠诚角色在接近策反条件时，可能触发"背水一战"：

- 角色意识到自己在被策反 → 主动向队伍求助
- 如果求助成功 → 忠诚度反弹，全队凝聚力回升
- 如果求助被玩家拦截（撤回/已读不回）→ 角色陷入更深的绝望

策反路线的最后一步往往是最危险的——你必须在角色"最后的求救"到达队友之前将其拦截。

### 7.5 胁迫路线的道德后果

C（胁迫）是四条路线中最"脏"的——你在威胁一个可能无辜的人。系统必须确保胁迫**不是最优解**：

- 被胁迫的资产反噬概率最高（每回合 8-12%，远高于其他类型的 1-3%）
- 被胁迫的角色会在后续季度以"复仇者"身份回归，带着专门针对你的能力
- 共情线结局中，玩家需要面对自己胁迫过的角色的控诉——这是道德选择最有力量的时刻
- **设计原则**：让玩家**可以**做坏事，但让他们**感受到代价**

### 7.6 策反的系统意义

策反角色在后续季度中可以：

- 以在编资产身份留在下一季度的队伍中（如果该角色转队）
- 以 NPC 形式回归，提供情报、特殊卡牌或组织权限
- 改写特定事件的结果
- 降低下一季度的初始可疑度（"因为你有内线"）

策反不是额外奖励，而是影响战役分支与结局的核心系统。

## 八、双面立场系统 (Dual Stance System)

**V3.1 核心新增系统。**

### 8.1 核心概念

本作的玩家不只是"魔王的工具"。在操纵勇者的过程中，玩家会对某些角色产生真实的情感——也许是牧师的隐忍让你心疼，也许是盗贼的坦率让你共鸣。

当这种共情发生时，玩家应该有权选择**用同一套工具去保护这个角色，而不只是摧毁她**。

但保护不是免费的善良。你是魔王军的 HR，你的老板要看到成果。**如果你开始保护勇者，你自己就变成了一个双面间谍。**

### 8.2 魔王忠诚度 (Loyalty Meter)

每个季度玩家有一个**魔王忠诚度计量器**（初始值 100）：

- 破坏操作 → 忠诚度上升或持平（魔王满意）
- 保护操作 → 忠诚度下降（魔王起疑）
- 中性操作（纯情报收集、不作为）→ 忠诚度缓慢下降（魔王认为你"不够积极"）

#### 忠诚度阶段效应

| 忠诚度区间 | 魔王态度 | 效果 |
|-----------|---------|------|
| 80-100 | 信任 | 全部权限可用，每季度获得额外经费储备奖励（+3） |
| 50-79 | 正常 | 标准状态 |
| 30-49 | 质疑 | 魔王开始派人监视你 → 某些操作（如渗透者指令、保护操作）有概率被魔王方发现 |
| 10-29 | 警告 | 魔王明确警告你"最近表现不佳" → 下一个汇报节点必须交出有说服力的成果，否则 Game Over |
| 0-9 | 清除 | 魔王判定你已叛变 → Game Over（被魔王军清除） |

#### 忠诚度的回复

- 成功制造一次重大事故 → +10-15
- 成功策反一名角色 → +20
- 在汇报节点提交令人满意的成果 → +10-20
- 牺牲一名你不在乎的角色 → +5-10

**核心张力**：你不能保护所有人。如果你想保护牧师，你可能需要把盗贼交出去作为"给魔王的成绩单"。**选择保护谁、牺牲谁**——这就是保护线自己的核心博弈。

### 8.3 汇报节点 (Report Events) — V4.0 重构为地图事件节点

**V4.0 关键变更**：汇报节点不再是回合内的额外行动，而是季度小地图上的**固定遭遇节点**。

#### 地图节点设计

每个季度的小地图上有 1-2 个标记为 📋 的汇报节点，与战斗遭遇、休整点、商店等并列。当玩家在地图推进到汇报节点时：

1. **暂停正常回合流程**——汇报有自己的专属 UI 界面
2. **进入汇报专属场景**——画面切换为"魔王通讯频道"，左侧显示魔王对话，右侧显示你本季度的行动摘要
3. **完成汇报后返回地图**——继续下一个节点

这使得汇报成为一个有**仪式感和紧张感**的独立体验，而不是塞在某个回合里的额外按钮。

#### 汇报选项

进入汇报节点后，玩家需要从以下选项中选择展示什么：

- **展示真实进度**：如果你确实在破坏队伍，正常汇报即可
- **伪造战报**：如果你在暗中保护，你需要编造一份看起来合理的"破坏成果"
  - 消耗 1 张伪造卡 + 2 行动力
  - 有失败概率（取决于你实际制造了多少"可见的"破坏）
  - 如果伪造被识破 → 忠诚度 -30
- **牺牲棋子**：主动让一个你不在乎的角色遭受重大打击，作为交给魔王的"成果"
  - 忠诚度回复，但该角色承受真实伤害
  - 如果该角色恰好是你保护对象的朋友……你需要面对后果

#### 注意力层级：不占用回合

汇报节点作为地图事件，**完全不占用正常回合的前台决策空间**。玩家在抵达汇报节点前可以看到它在地图上的位置，提前规划——"我需要在汇报之前制造至少一次可见的事故"或"我需要在汇报之前存一张伪造卡"。

这种"可预见的大考"设计让汇报从认知负担变成了**策略目标**——它不是每回合都要操心的事，而是你朝着它规划的里程碑。

汇报节点是保护线中最紧张的时刻——你需要用"反向社交工程"来欺骗你的上级，就像你用社交工程来破坏勇者一样。

### 8.4 保护线的节奏设计

#### Q1：强制破坏（教学阶段）

- Q1 不提供保护选项
- 玩家必须完整体验一次"亲手瓦解一支队伍"
- 这确保了两件事：
  1. 玩家学会了所有破坏工具的用法
  2. 玩家在 Q2 选择保护时，知道自己在放弃什么——这不是"从来没做过坏事所以想做好人"，而是"我做过坏事，我知道那是什么感觉，现在我选择不再继续"

#### Q2+：保护选项逐渐浮现

- Q2 开始，当玩家对某个角色的了解达到一定深度（通过深度扫描或频繁监听私聊）时，档案视角中会开始展示该角色更多的个人故事、梦想和恐惧
- 系统不会弹窗问"你要保护她吗？"——玩家只需要开始**用卡牌做保护操作**，系统就会自然地调整忠诚度和叙事走向
- 保护不是一个"选了就锁定"的模式切换，而是每回合每张牌的方向选择的累积

### 8.5 保护线的三种层级

| 层级 | 描述 | 忠诚度影响 | 叙事走向 |
|------|------|-----------|---------|
| **消极保护** | 不主动伤害某角色（但也不积极保护） | 轻微下降 | 角色自生自灭，你只是"没有出手" |
| **积极保护** | 用卡牌反向操作来修复该角色的困境 | 显著下降 | 角色在你的暗中帮助下逐渐好转；你需要在汇报中掩盖痕迹 |
| **双面间谍** | 系统性地保护整支队伍或多名角色，同时对魔王伪装成还在破坏 | 急剧下降 | 完全反转立场，走向共情线或第三路线结局 |

### 8.6 保护与 MICE 策反的交互

保护和策反不是互斥的——它们可以产生复杂的交织：

- 你可以**策反一个角色后让 TA 帮助你保护另一个角色**（给资产下达"暗中保护 X"的指令）
- 你可以**先保护一个角色、建立信任，然后利用这份信任进行 E 型策反**——这是最优雅也最残忍的路线
- 你可以**用 C 型胁迫策反一个角色，然后在后续季度中选择帮助 TA 摆脱胁迫**——从施害者变成拯救者

这些交织不需要额外系统——它们是 MICE + 双面立场自然产生的涌现玩法。

## 九、角色持续化 (Persistent Characters)

### 9.1 持续化深度

角色跨季度保留以下完整档案：

- 名字与外貌
- 职业与性格标签
- 完整人格倾向
- 关键关系网络
- 创伤记录（"Q1 曾被队友抛弃"）
- 履历标签（"曾参与 Q2 远征"、"被甩锅两次"）
- **人格演变**（V3.0 新增）：经历过的事件会永久修改角色人格——例如被反复甩锅的角色可能从"忠诚"变为"多疑"

### 9.2 季度结算状态

| 状态 | 描述 | 后续可能 |
|------|------|---------|
| 战死 | HP 归零 | 永久退场，可能成为其他角色的创伤来源 |
| 重伤退役 | 战斗致残 | 可能在后续季度以非战斗 NPC 回归 |
| 存活转队 | 加入新小队 | 带着旧创伤和关系进入新队伍 |
| 被招安 | 加入魔王阵营 | 后续季度提供资源/情报/改写事件 |
| 精神崩溃 | 压力过高退出 | 可能在后续季度以"复仇者"身份回归 |
| 叛逃 | 独自离开冒险体系 | 可能在终局作为独立势力出现 |

### 9.3 叙事作用

- Q1 的情感积累在 Q2/Q3 有回报
- 玩家对"自己造成的后果"形成长期记忆
- 游戏不是重复开局，而是推进一段组织史与人物命运
- **再次遭遇已知角色时，玩家拥有信息优势**（知道其人格弱点和创伤），但角色也可能因为过往经历变得更难对付

### 9.4 角色情报拼图系统 (Character Intel Jigsaw) — V4.2 核心新增

**V4.2 核心新增系统。** 与雾灯模型（4.3）配合，构成情报层的完整设计。

每个角色不是一个"属性面板"，而是一张**不完整的拼图**。玩家通过不同频道、不同时机、不同条件逐块补齐，逐步从"模糊的印象"走向"精确的画像"。

#### 拼图碎片的四种类型

| 类型 | 获取方式 | 示例 | 可见性 |
|------|---------|------|--------|
| **公开碎片** | 免费，角色登场即可见 | 职业、外貌、公开性格标签、群聊中的表面态度 | 拼图上始终为 ■ |
| **情报碎片** | 消耗 AP 解锁（私聊全文 / 深度扫描） | 隐藏矛盾、精确压力阈值、内心渴望 | 花 AP 后从 □ 变为 ■ |
| **时机碎片** | 仅在特定条件窗口内可发现（Context-Specific） | 深夜营火谈话揭露童年创伤、压力爆发时的内心独白 | 条件满足且花 AP 监听时 □→■；窗口过后仍为 □ |
| **深层碎片** | 需要多个前置碎片已解锁才会出现 | 真正的 MICE 策反路线、最深层恐惧、人生终极目标 | 前置满足后在下次深扫时 □→■ |

#### 时机碎片的触发窗口设计

时机碎片是整个系统中最有设计价值的部分——它们只在特定游戏状态下出现，创造"在正确的时间监听正确的对话"的情报官体验：

**触发条件示例**：

| 触发条件 | 时机碎片内容 | 窗口持续 |
|---------|------------|---------|
| 牧师压力 > 70 且盗贼在场 | 牧师和盗贼在营火旁 deep talk，谈到被师门抛弃的过去 | 1-2 回合 |
| 战士和法师公开决裂后 | 法师在私聊中向盗贼坦白"我其实想离开这支队伍" | 当回合 |
| 某角色在战斗中被队友误伤 | 受伤角色的内心独白揭示对"背叛"的恐惧根源 | 当回合 |
| 成功用 E 型接触目标 3 次以上 | 目标在深扫中主动展露更深层的自我 | 持续 |

**设计规则：关键情报的冗余路径**

为防止"错过一个窗口就进入死路"，关键战略情报（如 MICE 路线是否可行/锁死）必须有多条发现路径：

| 情报重要性 | 发现窗口数量 | 逻辑 |
|-----------|------------|------|
| **关键战略**（MICE 路线可行/锁死） | 至少 2-3 个不同窗口 | 不同条件触发同一结论，避免"一次错过就死路" |
| **重要战术**（精确触发阈值） | 至少 1-2 个窗口 | 可以只有一个，但免费层摘要必须给强烈暗示（🔵 热度信号） |
| **档案/风味**（童年创伤、深层恐惧） | 可以只有 1 个窗口 | 错过不影响通关，但增加重玩动力 |

**冗余路径示例——"牧师可被 E 型策反"这条情报**：

- **路径 A**（回合 3 营火 deep talk，时机碎片）：最早、最完整——听到完整故事，同时解锁"E 型可行"和"C 型锁死"
- **路径 B**（回合 5 牧师压力爆发时深度扫描，情报碎片）：内心独白暗示"我只想被认可"——推断 E 型可行
- **路径 C**（尝试 C 型胁迫后的系统反馈）：牧师剧烈反抗 + 拼图更新"MICE-C：锁死 ✗"——反向发现

路径 A 是奖励主动侦察的"最优发现"。路径 B 和 C 是兜底。

#### 拼图碎片的机械效果

碎片不只是叙事风味——它们直接影响游戏机制：

| 碎片内容 | 机械效果 |
|---------|---------|
| "可被 M（金钱）策反" | MICE-M 进度条解锁，可以开始培养 |
| "绝对不可被 C（胁迫）策反" | MICE-C 路线标记为 ✗，防止玩家浪费资源 |
| "最深层恐惧：蜘蛛" | 如果环境操作卡中有"虫群出没"，对该角色效果翻倍 |
| "人生目标：证明师门是错的" | 如果玩家帮助其达成目标（保护线），忠诚度回报极高 |
| "童年创伤：被权威抛弃" | 甩锅操作对该角色的压力增加 +50%（踩中创伤） |
| "和盗贼有隐秘好感" | 利用盗贼作为中间人的操作效果翻倍 |

#### 情报拼图面板 UI

每个角色的情报面板是一张视觉化拼图：

```
┌─────────────────────────────────────┐
│  [牧师] 情报档案       完成度 7/15   │
├─────────────────────────────────────┤
│ ■ 公开身份：治疗师，队内新人          │  ← 免费
│ ■ 性格标签：忠诚 + 自卑             │  ← 免费
│ ■ 压力状态：高（72/100）            │  ← 免费（外在观察）
│ ■ 队内关系：被战士施压               │  ← 1 AP（私聊解锁）
│ ■ 内心渴望：被认可                  │  ← 2 AP（深扫解锁）
│ □ [未知] ？？？                    │  ← 未发现
│ □ [未知] ？？？                    │  ← 未发现
│ ■ MICE-E：可行 ✓                  │  ← deep talk 解锁
│ ■ MICE-C：锁死 ✗                  │  ← deep talk 解锁
│ □ MICE-M：？                      │  ← 未发现
│ □ MICE-I：？                      │  ← 未发现
│ ◇ [时机碎片] 需要特殊条件            │  ← context-specific
│ ◇ [时机碎片] 需要特殊条件            │  ← context-specific
│ □ [深层恐惧] ？                    │  ← 未发现
│ □ [人生目标] ？                    │  ← 未发现
└─────────────────────────────────────┘
```

- **■** = 已发现（点击可查看详情）
- **□** = 已知存在但未发现（给玩家"还有多少没找到"的感知）
- **◇** = 时机碎片（有特殊触发条件，满足时会高亮闪烁提示）
- **完成度 7/15** = 收集进度（驱动完成主义者，提供重玩动力）

#### 跨季度持续化

角色情报拼图是**人事档案 (Dossier)** 跨季度资源的具体载体：

- 已解锁的碎片永久保留在档案中
- 当同一角色在后续季度再次出现时，已有碎片直接可见（不需要重新解锁）
- 部分碎片可能因角色经历了新事件而**更新或变化**（如"被甩锅两次后，从'忠诚'变为'多疑'"→ 旧碎片标记为过期，新碎片出现）
- 新季度可能出现新的时机碎片（角色有了新的故事和关系）

#### 与五层文本架构的关系

角色情报拼图不是额外的内容负担——它直接对接五层文本架构（第十二章）：

| 拼图碎片类型 | 对应文本层 | 内容来源 |
|------------|----------|---------|
| 公开碎片 | 第一层（设计减量） | 角色登场 UI 标签和简介 |
| 情报碎片 | 第二层（消息 DNA） | 私聊全文来自消息家族，深扫独白来自人格变体 |
| 时机碎片 | 第四层（手写锚点） | Context-specific 的关键场景手写文本 |
| 深层碎片 | 第四层（手写锚点） | MICE 策反对话和角色终极揭示 |

#### 情报蓝图 (Intel Blueprint)

每个角色在设计阶段需要制定一份**情报蓝图**，规划全部碎片及其发现路径。MVP 阶段每角色约 12-15 块碎片：

- 公开碎片：3-4 块（免费可见）
- 情报碎片：4-5 块（AP 解锁）
- 时机碎片：2-3 块（context-specific）
- 深层碎片：2-3 块（需前置条件）

6-8 个 MVP 角色 × 12-15 块 = 总计 72-120 块碎片。

## 十、玩家成长 (Player Growth)

### 10.1 卡组构筑成长

每个季度推进过程中，玩家通过地图节点获得：

- 新卡（选 1/3 形式）
- 删牌机会
- 升级牌机会（普通 → 精制 → 大师级）

构筑重点不是单卡强度，而是围绕目标流派组装 combo 配件。

### 10.2 权限成长

HR 权限按季度逐步解锁，从根本上扩大操作空间：

| 季度 | 解锁权限 |
|------|---------|
| Q1 | 基础监听、撤回、延迟、放大、泄露 |
| Q2 | 伪造、渗透者系统、私聊窃听 |
| Q3 | 禁言、审计、绩效篡改、双渗透者 |
| Q4 | 人事调动、通讯全面管制、深度伪造 |

### 10.3 情报成长

- 玩家积累老兵档案、创伤记录与关系史
- 同一角色再次出现时，深度扫描的行动力消耗降低（你已经了解这个人，情报拼图中已解锁的碎片直接可见——人事档案生效）
- 跨季度的情报积累让玩家真正感受到"越来越专业"

### 10.4 资源成长体系 (V4.1)

#### 行动力成长

| 来源 | 行动力增量 |
|------|-----------|
| 季度推进（基础增长） | Q1: 3 AP → Q2: 3 AP → Q3: 4 AP → Q4: 4 AP |
| 特殊成就（如"全季度杰作评价"） | 永久 +1 AP（极稀有） |

#### 经费储备成长

| 来源 | 储备增量 |
|------|---------|
| 季度推进（基础增长） | Q1: 8 → Q2: 10 → Q3: 12 → Q4: 15 |
| 上季度魔王信任 ≥ 80 | +3 额外储备 |
| 上季度招安成功 | +2 额外储备（策反资产提供后勤支持） |
| 上季度低暴露通关 | +2 额外储备（组织信任你的能力） |

#### 情报成本降低

| 来源 | 效果 |
|------|------|
| 人事档案（跨季度积累） | 已知角色的深度扫描从 2 AP 降至 1 AP |
| 招安资产提供内线情报 | 资产所在队伍的私聊监听免费（不消耗 AP） |
| 权限升级 | 解锁新的情报频道和卡牌使用权 |

## 十一、核心机制模块 (Key Systems)

### 11.1 性格标签弱点系统

角色由 `[职业] + [性格A] + [性格B] + [关系状态]` 构成。

六种基础人格：

| 人格 | 弱点 | 被激活条件 | 激活后行为 |
|------|------|-----------|-----------|
| **傲慢** | 无法接受质疑 | 权威被挑战、功绩被否认 | 暴怒发言、拒绝合作、主动产生冲突节点 |
| **多疑** | 容易误读信息 | 私聊泄露、消息被篡改、异常行为 | 开始质疑队友、自行调查（可能发现你的操纵） |
| **贪婪** | 利益诱惑 | 资源分配不公、发现额外战利品 | 脱离团队去追逐私利、产生资源冲突节点 |
| **暴躁** | 压力溢出即爆发 | 连续受挫、被指责、被忽视 | 公开攻击队友、拒绝指挥、可能误伤 |
| **自卑** | 容易被甩锅击溃 | 事故责任被归到自己身上 | 自我怀疑、治疗/支援延迟、压力飙升 |
| **忠诚** | 背叛后反噬最大 | 信任的人让自己失望 | 长期沉默 → 突然爆发、可能转为"叛逃"或"复仇" |

### 11.2 人格交互矩阵 (Personality Matrix)

**V3.0 新增。** 人格之间不只是各自独立响应事件，还会**互相催化**：

| A 的人格 | B 的人格 | 互动效果 |
|----------|----------|---------|
| 傲慢 | 傲慢 | 必然产生权力争夺，队内会自然分裂为两个派系 |
| 傲慢 | 自卑 | 傲慢者习惯性地命令和忽视自卑者，自卑者压力持续累积 |
| 多疑 | 多疑 | 互相猜忌形成恶性循环，一旦出现任何异常信息即失控 |
| 贪婪 | 忠诚 | 贪婪者的自私行为会直接激怒忠诚者，忠诚者可能主动举报 |
| 暴躁 | 自卑 | 暴躁者的怒火直接砸在自卑者身上，自卑者崩溃最快 |
| 忠诚 | 傲慢 | 忠诚者倾向服从傲慢者（只要不被背叛），形成稳固同盟 |

**策略意义**：
- 队伍的人格组合本身就预设了"最佳攻击路线"——识别出最容易互相催化的人格对，是每回合侦察阶段的核心判断
- 有些队伍天然脆弱（傲慢队长 + 自卑牧师），有些天然坚韧（忠诚队长 + 忠诚副手），后者需要更精妙的策略

### 11.3 团队凝聚力系统

凝聚力代表小队是否还能作为一个"组织"继续运作。

影响因素：
- 误解 / 背锅 / 资源分配不公 / 指令冲突（降低）
- 队友死亡 / 长期低信任关系（大幅降低）
- 成功完成战斗 / 背书 / 安抚 / 团建（回升）

凝聚力有**阶段效应**：

| 凝聚力区间 | 队伍状态 | 效果 |
|-----------|---------|------|
| 80-100 | 团结 | 角色主动互相背书，修复速度快 |
| 50-79 | 正常 | 标准状态 |
| 30-49 | 紧张 | 角色开始消极配合，小矛盾容易升级 |
| 10-29 | 危机 | 角色可能拒绝执行命令，自发冲突频率大增 |
| 0-9 | 解散边缘 | 任何事故都可能触发解散 |

### 11.4 动态声望与公会应对

玩家长期偏好某种流派时，王国公会会调整派遣队伍结构：

- 如果玩家常用离间策略 → 后续队伍可能派遣关系更紧密的老搭档
- 如果玩家常用压力战 → 后续队伍可能配备心理辅导角色
- 如果玩家频繁招安 → 后续队伍可能加入专职反间角色

不是给角色硬免疫，而是从源头改变来者是谁——迫使玩家跨季度调整卡组。

### 11.5 勇者反情报升级 (Counter-Intelligence Escalation)

**V3.0 新增。** 勇者公会不是静态的对手，而是一个会**学习和适应**的组织。

#### 季度间的公会适应

| 季度 | 公会应对 |
|------|---------|
| Q1 后 | 意识到有人在干扰通讯 → Q2 队伍配备基础加密 |
| Q2 后 | 确认存在系统性监听 → Q3 队伍有反间训练 |
| Q3 后 | 掌握部分操纵手法 → Q4 队伍配备专职反间角色 |

#### 反间角色

Q3+ 可能出现的特殊角色：

- **通讯官**：可识别伪造消息（伪造操作对其有概率失效）
- **参谋**：可分析战斗事故模式（如果连续两回合都出现"恰好"的事故，参谋会提出质疑）
- **审计员**：可检查行政操作的痕迹（审计、绩效篡改等操作的暴露值翻倍）

### 11.6 固定事件节点

每个季度包含固定触发点：

- 某成员可能战死
- 某成员可能升职或转队
- 某起事故会成为下季度的舆论事件
- **关键抉择点**（V3.0 新增）：某些事件会强制玩家在两个互斥选项中选择（例如："这次事故你把锅甩给牧师还是盗贼？"两者都有后续影响，不存在无代价选项）

设计原则：**节点固定，结果可变。**

## 十二、文本生产管线 — 五层防线架构 (Text Production Pipeline)

**V4.0 核心新增。** 群聊文本是本作的"地基"——所有系统设计都建立在"读群聊本身有趣"这个前提上。如果文本质量不过关，上面的一切都是空中楼阁。

本章提供一个完整的生产方案，将"不可能的文本量"拆解为可管控的工作流。

### 12.1 问题量化

| 维度 | 需求规模 |
|------|---------|
| 角色数 | 每季度 4-5 人，跨 4 季度 ≈ 6-8 个有深度的持续角色 |
| 人格类型 | 6 种基础人格 × 人格演变 |
| 回合数 | 每季度 ~10 回合 × 4 季度 = ~40 回合 |
| 每回合消息量 | 群聊 5-10 条 + 私聊 2-5 条 + 战后余波 3-5 条 |
| 分支因子 | 玩家操作导致的不同情绪/关系状态 → 每条消息需 3-5 个变体 |
| **总文本量估算（朴素方案）** | **40 回合 × 15 条/回合 × 4 变体 ≈ 2,400 条独立消息** |

2,400 条消息，每条需要角色声音一致、情绪准确、携带策略信息、不重复——这对独立团队是灾难性的工作量。

核心判定标准：**玩家读到第 30 回合的群聊时，是否仍感受到"一群真实的人在说话"**，还是已经开始跳过文字只看数值。

**解决方案不是一个银弹，而是五层叠加的防线**——每层缩小问题规模、提升质量下限，直到问题从"不可能"变成"可管控"。

### 12.2 第一层：设计层减量 — 让问题变小

在动笔写任何文字之前，从设计层面**大幅压缩需要写的文本量**。

#### 12.2.1 短消息格式 (Messaging App, Not Novel)

群聊不是小说对话——是**即时通讯软件**。真实的群聊消息：

```
[战士] 我去前排
[牧师] 收到
[盗贼] 上次就是你"去前排"然后我被集火的
[战士] ???那是你站位的问题
[法师] 能别吵了先打
```

**不是**：

```
[战士] "各位，我认为这次战斗我应该承担前排坦克的职责，
       毕竟上次我们的阵型部署出了一些小问题..."
```

**短消息（每条 5-15 字）= 写作难度降低 80%，角色声音更容易区分。**

#### 12.2.2 非文字信息载体

真实的群聊不只有文字——大量信息通过**非文本元素**传递：

| 非文字元素 | 携带的游戏信息 | 文本量 |
|-----------|---------------|--------|
| 已读/未读状态 | 谁在关注谁、谁在冷战 | 0 |
| "输入中..." 气泡 | 某人犹豫了很久又删掉了（暗示内心挣扎） | 0 |
| 表情/贴纸反应 | 情绪快速表达（👍😤🙄💔） | 0 |
| [语音消息 0:23] | 某人不想留下文字证据 / 情绪太激动打不了字 | 0 |
| 消息撤回痕迹 | "[战士] 撤回了一条消息" | 7 字 |
| 在线/离线状态 | 谁主动退出对话、谁深夜独自上线 | 0 |
| 群聊名修改记录 | "群名已改为：废物小队 → 远征第三小队" | 极少 |

**零文本元素可传递 30-40% 的游戏信息量。** 不仅节省文本，还为玩家提供了额外的"阅读空间"——一个 "输入中..." 的气泡消失了，比任何台词都更能暗示犹豫和压抑。

#### 12.2.3 信息密度分级

群聊消息按功能分三级，只有最高级需要精写：

| 级别 | 功能 | 质量要求 | 占比 | 示例 |
|------|------|---------|------|------|
| **A 级** | 携带关键策略信息的"可操作节点" | 必须精写 | ~20% | "下一场我不给你加盾了，自己看着办" |
| **B 级** | 烘托气氛、展示关系的"背景噪音" | 模板 + 变量即可 | ~50% | "到了" / "收到" / "随便你" / "😤" |
| **C 级** | 纯功能性系统消息 | 固定文本 | ~30% | "[系统] 战术频道已加密" / "[已读]" |

**减量效果**：2,400 条 → 实际需要精写 ≈ **480 条 A 级消息**。仍不少，但已进入"人力可控"区间。

### 12.3 第二层：结构化写作 — 消息 DNA 系统

不是写 480 条独立的台词，而是写 **"消息家族 (Message Family)"**——每个家族是一个语义模板，由多个人格变体组成。

#### 消息 DNA 构成

```
意图 (Intent) × 情绪 (Emotion) × 人格 (Personality) × 上下文 (Context)
```

#### 示例：消息家族 "拒绝配合"

| 意图 | 情绪 | 人格 | 输出 |
|------|------|------|------|
| 拒绝配合 | 愤怒 | 傲慢 | "你没资格指挥我" |
| 拒绝配合 | 愤怒 | 暴躁 | "滚" |
| 拒绝配合 | 疲惫 | 自卑 | "...我做不到" |
| 拒绝配合 | 冷漠 | 多疑 | "你确定你的计划没问题？我先观望" |
| 拒绝配合 | 受伤 | 忠诚 | "我以为我们是一个团队" |
| 拒绝配合 | 贪婪 | 贪婪 | "那边有个宝箱，我先去拿" |

一个消息家族写 6 条变体（对应 6 种人格），覆盖一个完整的意图 × 情绪组合。变体之间**只有措辞不同，携带的策略信息完全相同**——"此角色正在拒绝配合"。

每条变体只有 5-15 字，整个家族的写作时间 ≈ 10 分钟。

#### 消息家族需求估算

| 意图类别 | 示例意图 | 估算数量 |
|----------|---------|---------|
| 战斗协调 | 请求支援、分配战位、报告状态 | ~15 |
| 关系互动 | 抱怨、安慰、背书、质疑、表白立场 | ~20 |
| 冲突升级 | 甩锅、公开指责、冷战、威胁退队 | ~15 |
| 应激反应 | 恐慌、崩溃、暴怒、沉默、自我怀疑 | ~15 |
| MICE 相关 | 对金钱的渴望、理念动摇、被胁迫的恐惧、自尊受损 | ~15 |
| **总计** | | **~80 个消息家族** |

**80 家族 × 6 变体 = 480 条变体**，每条 5-15 字，总文本量约 5,000-7,000 字——**一个资深文案 1-2 周的工作量**。

### 12.4 第三层：运行时组装 — 对话编排引擎

消息家族写好后，需要在运行时组成**读起来自然的对话流**。

#### 12.4.1 对话序列模板 (Conversation Choreography)

不随机抽取消息拼接，而是预定义**对话节奏模板**：

```
[模板: 战后事故追责]

1. [触发者] 提出质疑      → 消息家族 "质疑行动"
2. [被质疑者] 反驳        → 消息家族 "自辩"
3. [旁观者A] 附和一方     → 消息家族 "站队"
4. [旁观者B] 试图调停     → 消息家族 "调停"
5. [触发者] 升级或放弃    → 根据压力值选择 "升级指控" 或 "勉强接受"
```

模板确保：
- 对话有**起承转合**的节奏（不是随机拼接）
- 不同角色的发言有**逻辑顺序**
- 同一模板填入不同消息家族，每次玩的具体台词都不同

#### 对话序列模板需求估算

| 场景类型 | 估算数量 |
|----------|---------|
| 战前规划 | ~5 种模板 |
| 战后复盘（正常/事故/重大事故） | ~8 种模板 |
| 日常互动（闲聊/倾诉/争吵） | ~10 种模板 |
| 关键事件（背叛揭露/策反对话/崩溃/汇报） | ~12 种模板（手写） |
| **总计** | **~35 个对话序列模板** |

每个模板 5-8 步——是结构化工作，不是创意写作。

#### 12.4.2 上下文感知插值

对话编排引擎在选择消息变体时考虑当前状态：

```
选择逻辑：
1. 当前角色人格 → 筛选人格变体
2. 当前情绪值   → 选择最匹配的情绪等级
3. 关系状态     → 调整称呼和语气（亲密度高用昵称，低用姓氏/职称）
4. 最近事件     → 注入具体指代
```

第 4 步是关键——**具体指代**是让模板文本不感觉"模板化"的核心技巧：

- 模板：`"{角色}在{地点}搞砸了{事件}，现在还有脸说话？"`
- 实例：`"你在补给仓库搞砸了治疗协调，现在还有脸说话？"`

角色名、地点、事件都是运行时变量，但句式的"人味"来自模板本身。

#### 12.4.3 非文字元素的运行时编排

非文字元素（12.2.2）也需要编排逻辑：

| 情境 | 自动触发的非文字行为 |
|------|-------------------|
| 角色压力 > 70% | 回复延迟增加、偶尔出现 "输入中..." 后消失 |
| 两角色信任 < 30 | 互相的消息不再显示"已读" |
| 角色被孤立 | 该角色的在线状态变成"离线"或"隐身" |
| 争吵升级 | 双方消息频率暴增，旁观者变为只发表情 |
| 角色崩溃前兆 | 消息变短、用词变简单、标点消失 |

这些行为不需要文字，但能极大提升"真实感"。

### 12.5 第四层：关键场景手写 — 20% 精写覆盖 80% 记忆

有些时刻**必须手写**——它们是玩家的情感记忆锚点：

| 场景类型 | 为什么必须手写 | 预估数量 |
|----------|---------------|---------|
| **MICE 策反对话** | "推销你的理由"时刻，必须让玩家纠结 | 4 MICE × 8 角色 = ~32 段 |
| **角色崩溃独白** | 压力过高的情绪爆发，游戏最有冲击力的文本 | 8 角色 × 2 阶段 = ~16 段 |
| **汇报节点对话** | 玩家与魔王的"反向社交工程"，需要紧张感 | 4 季度 × 3 选项 = ~12 段 |
| **结局叙事** | 玩家行为的最终总结 | 5 结局 × 1 段 = ~5 段 |
| **Q1 教学叙事** | 第一印象，决定玩家是否继续 | ~10 段关键对话 |
| **总计** | | **~75 段手写文本** |

每段 50-200 字。这是真正的创意写作任务，需要最好的文案，但规模完全可控。

**关键原则：手写文本 = 玩家记住的文本。** 没人会记住第三回合的战术闲聊，但每个人都会记住牧师崩溃那段独白。

### 12.6 第五层：验证管线 — "文本味觉测试"

前四层解决"怎么写"，这层解决"怎么知道写得够不够好"。

#### 12.6.1 最小文本原型 (Text Taste Test)

**在任何系统代码之前完成此验证**，成本极低（一个下午），但能避免数月无效开发：

```
步骤 1：写 Q1 第一场遭遇的 3 回合完整群聊脚本（约 50-80 条消息）
步骤 2：找 5 个从未接触本项目的人
步骤 3：只让他们读群聊文本（不告诉这是游戏，不展示任何系统）
步骤 4：问三个问题：
         ① "你想继续看下去吗？"（基础吸引力）
         ② "你觉得哪个角色最有意思？"（角色区分度）
         ③ "你觉得接下来会发生什么？"（悬念感）
步骤 5：判定
         → 3/5 回答"想继续" = 文本基线合格，可以继续开发
         → <3/5 = 文本方案需要重做，暂停系统开发
```

#### 12.6.2 重复度衰减审计

每写完一个季度的文本后跑审计：

```
统计消息家族的使用频率
→ 找出被调用 >5 次的家族
→ 为高频家族补充额外 2-3 条变体
→ 目标：玩家在一个季度内不会看到完全相同的台词
```

#### 12.6.3 角色声音一致性检查

| 检查项 | 方法 |
|--------|------|
| 同一角色在不同情绪下是否像"同一个人" | 把同一角色所有变体排在一起读 |
| 不同角色在同一情绪下是否能区分 | 遮住角色名，看能否猜出谁说的 |
| 人格演变后的声音是否自然过渡 | 按时间线排列同一角色发言，检查"忠诚→多疑"的演变是否突兀 |

#### 12.6.4 "跳过率"监测（正式版）

如果进入正式开发，在 Playtest 中追踪：
- 玩家阅读群聊的平均时间 vs 跳过（快速滚动）的频率
- 如果 A 级消息被跳过率 > 30%，说明文本质量或信息密度有问题
- 如果 B 级消息被全部跳过，说明背景噪音太多需要减量

### 12.7 工作量总览

| 层 | 工作内容 | 工作量 | 负责方 |
|----|---------|--------|--------|
| 第一层 | 非文字元素设计 + 信息分级定义 | 1 周 | 策划 |
| 第二层 | 80 个消息家族 × 6 变体 = 480 条短消息 | 1-2 周 | 文案 |
| 第三层 | 35 个对话序列模板 + 编排引擎设计 | 2 周 | 策划 + 程序 |
| 第四层 | 75 段手写关键文本 | 2-3 周 | 资深文案 |
| 第五层 | 验证测试（贯穿始终） | 持续进行 | 全团队 |
| **总计** | | **6-8 周（单人）** | |

**核心结论**：通过五层防线叠加，"文本质量"从一个无解的致命风险变成了一个**可估算、可拆分、可验证的生产任务**。最重要的一步是第五层的 Text Taste Test——它必须在第二层写作之前完成，用来验证"短消息群聊格式"本身是否足够有趣。如果验证失败，说明核心体验假设有误，需要回到产品层面重新思考，而不是继续堆系统。

## 十三、多结局结构 (Endings)

### 13.1 忠诚线：魔王提拔结局

- 玩家始终忠于魔王利益（忠诚度全程 ≥ 80）
- 高效瓦解或消灭所有小队
- 最终获得魔王赏识，晋升为核心干部
- 变体 A（"工具"结局）：全程使用粗暴方式，魔王评价你是"好用的工具"——升职但不受尊敬
- 变体 B（"战略家"结局）：全程使用精妙方式，魔王视你为"真正的战略家"——成为核心智囊

### 13.2 共情线：双面间谍结局

- 玩家在某个季度开始对某位角色使用保护操作（忠诚度降至 50 以下）
- 需要在保护目标与维持魔王掩护之间持续周旋——用"反向社交工程"欺骗自己的上级
- 变体 A（"守护者"结局）：成功保护目标角色存活并完成任务 → 你在阴影中看着她走向光明，而你留在黑暗里
- 变体 B（"叛逃"结局）：保护失败或掩护被识破 → 你和被保护的角色一起被追杀，但至少你们互相知道真相
- 变体 C（"赎罪"结局）：你曾用 C（胁迫）策反过某个角色，后续季度中你选择帮助 TA 摆脱胁迫 → 从施害者变成拯救者

### 13.3 第三路线：制度反叛结局

- 玩家既不忠于魔王，也不完全帮助勇者
- 利用掌握的情报和人脉（策反的资产 + 存活勇者 + 公会内部关系），建立独立于两方的第三势力
- 最终目标：瓦解魔王军与勇者公会的二元对立，建立新秩序
- 触发条件：至少在两个季度中达成"精妙"以上优雅度，策反 ≥ 2 名角色（含至少 1 名 I 型理念资产），且忠诚度在 30-70 区间内（两边都不完全信任你）

### 13.4 隐藏结局：完美人偶师

- 全 Campaign 优雅度均为"杰作"
- 没有角色死亡（全部通过解散或招安结束）
- 可疑度从未超过 Level 1
- 结局：没有人知道你存在过。勇者们认为是自己的命运，魔王认为是天意。你站在阴影中，看着自己编排的剧本完美落幕。

### 13.5 隐藏结局：镜像反转

- 全 Campaign 中至少使用 3 次 C（胁迫）策反
- 最终季度中，所有被胁迫的资产同时反水，联合勇者对你发起清算
- 你以为自己是操纵者，实际上你才是被最终审判的那个人
- 触发条件极为苛刻（需要同时满足多个胁迫策反 + 忠诚度极高 + 从未保护过任何人）
- 这是对"纯恶路线"的最终叙事惩罚：工具人最终被自己打造的工具反噬

- **档案情报**：内心独白"也许……我不适合做冒险者。也许哪里都不需要我。"
- **📋 情报拼图更新**：[牧师] □→■ "可招安前置：85%"、◇→■ 时机碎片"自我放弃独白"解锁

**系统节点：**

- `威胁：战士 → 全队 [停止保护]`
- `请求：牧师 → 全队 [暂停推进 / 休整]`
- `抱怨：法师 → 战士 [嘲讽权威]`
- `资源冲突：盗贼 → 全队 [预占战利品归属]`

**玩家决策：终局三选一**

此时玩家面临一个真正的**战略分叉**：

- **A. 解散路线**：让战士彻底爆发，推动公开决裂 → 凝聚力归零 → 解散
- **B. 招安路线**：保护牧师的休整请求被否定的局面 → 让牧师最后的希望破灭 → 触发可招安 → 使用招安话术
- **C. 棋子化路线**：再推一把战士的暴躁 → 让战士变成棋子 → 战士自行摧毁队伍

每条路线需要不同的卡牌操作，且互斥。

**玩家选择 B（招安路线）：**

玩家从经费储备提取 2 AP（储备 7→5），本回合可用 AP = 3 + 2 = 5：情报 3 + 出牌 2。

1. `已读不回` 牧师的休整请求（费用 0 AP，暴露 +0）
   - 意图：让牧师的求救被全队无视，触发"自我放弃"
2. `人事背锅`，将下一次事故的责任预引导到牧师（费用 1 AP，暴露 +1）
   - 意图：确保战后余波中牧师再次成为甩锅对象
3. 对盗贼的资源冲突使用 `公开引用`（费用 1 AP，暴露 +1）
   - 意图：制造混乱，让牧师的求救彻底被淹没

本回合消耗：行动力 3（情报）+ 2（经费提取→出牌），暴露 +2（总可疑度：3 — 仍远低于 Level 1 的阈值 10），经费储备 7→5

**执行阶段：**

- 队伍带着未解决的矛盾进入房间
- 战士保护欲下降，牧师治疗犹豫，法师自保优先，盗贼走位偏向战利品
- 牧师因 `自我放弃` 状态，治疗优先级最低——自己被打到半血也不呼救

**因果链回放：**
> `[牧师请求休整被无视] → [牧师进入"自我放弃"状态] → [战斗中牧师下意识降低治疗优先级] → [战士受伤加重] → [战士怒骂牧师] → [人事背锅生效：事故责任再次归到牧师身上] → [牧师彻底动摇]`

**战后余波：**

- 战士："我受够了，这队谁爱带谁带。"
- 法师："那就别带了。"
- 盗贼："战利品我拿走，反正你们也配不上。"
- 牧师："……我们真的还要继续吗？"

**可招安窗口开启：**

牧师进入 `可招安` 状态。如果玩家有 `招安话术` 卡牌，可在此时使用：

> 系统生成一条来自"匿名渠道"的私聊消息：
> "你不需要继续为不感激你的人牺牲。有些地方会真正珍视你的能力。"

牧师（内心独白，深度扫描可见）：
> "……也许他们说的是对的。也许那里真的会不一样。"

**最终状态：**

- 团队凝聚力跌破阈值 → 队伍进入 `解散边缘`
- 牧师进入 `可招安` → 如果玩家使用招安话术，牧师脱离队伍加入魔王阵营
- 战士与法师进入 `公开敌对`
- 整季度总可疑度：3（远低于 Level 1）
- 优雅度评价：精妙 (Subtle)（3 回合仅使用 7 次操作，暴露极低，利用了大量角色自发冲突）

### Demo 验证结论

V4.2 推演证明了以下几点：

1. **雾灯模型让情报投资从盲赌变为有据判断**：免费摘要+热度信号（🔴🟡🟢🔵）让玩家在花 AP 前就知道哪个频道最可能有料——回合 1 的三个 AP 全部命中有价值目标
2. **三种情报货币消灭了"空包弹"**：每次 AP 投资都返回了战术/战略/档案情报中的至少一种——回合 2 的深扫同时获得了崩溃阈值（战术）、可招安种子（战略）、自我否定模式（档案）
3. **情报拼图创造了持续的发现进度感**：3 回合内牧师的拼图从 3/15 推进到 8/15，每次解锁都伴随"📋 拼图更新"的正反馈
4. **行动力统一资源池让"看 vs 做"成为核心张力**：信息获取有真实代价，但雾灯的免费层确保"不花 AP 也不完全盲目"
5. **经费储备提供了关键回合的爆发窗口**：回合 1 和 3 分别提取 1 AP 和 2 AP 完成关键操作
6. **暗线网络奖励优雅玩法**：回合 2 零暴露操作获得 +2 暗线积累
7. **操作链让回合之间产生了前后关联**：回合 1 的布置在回合 2-3 持续发挥作用
8. **连锁反应让回合 2 实现了零暴露推进**：角色自发冲突替代了玩家操作
9. **深度扫描让招安路线有了精确的触发时机**：不是碰运气，而是精准识别崩溃阈值后精确触发
10. **最终面临真正的战略分叉**（解散/招安/棋子化），而不是唯一最优解

### 15.2 文本生成方案

**V4.0 已在第十二章提供完整的五层防线架构。** 此处保留摘要：

## 十六、设计结论 (Current Direction)

《魔王的人力资源部》V4.2 的方向：

- 一个 **固定季度结构** 的战役型 Deckbuilding RPG
- 核心玩法是 **社交工程**：监听、分析、操纵、收割——或保护
- **每回合只需做 3 个决策**（注意力分层架构），其余系统在后台自动运行或仅在关键时刻弹出告警
- 每次操作都有 **代价与风险**（暴露系统 + 忠诚度系统），迫使玩家在激进与精妙之间、在破坏与保护之间权衡
- 卡牌之间有 **组合深度**（操作链三拍节奏），且每张卡支持**双面使用**（破坏/保护）
- **雾灯情报模型**：所有频道永远可见（免费摘要+热度信号），AP 买到的是精度而非访问权——新手依赖摘要做粗暴决策，高手通过精准投资做杰作级操作
- **三种情报货币**保证每次 AP 投资都有回报：战术情报（当场用）、战略情报（2-3 回合后用）、档案情报（跨季度积累进入情报拼图）
- **角色情报拼图 (Intel Jigsaw)**：每个角色是一张 12-15 块碎片的拼图，通过不同频道、不同时机逐块补齐——时机碎片创造"在正确的时间监听正确的对话"的情报官体验
- **情报获取和卡牌操作竞争同一个行动力池**（每回合 3 AP），"看什么"和"做什么"是永恒的取舍
- **经费储备**提供季度级经营维度：总量有限，选择在哪些关键回合爆发是中期策略核心
- **暗线网络**奖励优雅玩法：低暴露操作积累"政治资本"，可兑换零暴露操作——用过去的优雅购买未来的安全
- 勇者不是沙包，而是会 **自我修复、适应和反制** 的活组织
- 被操纵的角色会 **自发产生冲突**（连锁反应 / 棋子化），让高手能实现零暴露推进
- 同一目标有粗暴和精妙两种达成路径（**优雅度评价**），为重复游玩提供技巧追求
- 策反遵循 **MICE 四维模型**（金钱/理念/胁迫/自尊），是跨越多回合的情报经营过程
- 策反后的资产需要**持续运营和维护**，不同 MICE 路径产生质量迥异的资产
- 渗透者使用**策略设定模式**（设定一次、自动执行、关键分叉弹出），不挤占每回合的操作空间
- 汇报节点是**地图上的独立遭遇事件**，有自己的仪式感和专属 UI，不占用正常回合
- 玩家可以选择**保护某个角色**而非摧毁——但代价是成为双面间谍，需要同时欺骗魔王上级
- 角色面板默认展示**档案视角**（一个有故事的人），只有主动切换到情报视角才会看到 MICE 雷达（一个待攻破的目标）
- 群聊文本通过**五层防线架构**（第十二章）从致命风险变为可管控的生产任务：设计减量→消息DNA→编排引擎→手写锚点→验证管线
- **三层资源经济**（回合级行动力 / 季度级经费储备+暗线 / 跨季度人事档案+魔王信任+公会档案）让玩家像真正的情报主管一样管理有限资源
- 角色以名字、人格、关系、创伤、人格演变跨季度持续存在
- 多结局涵盖忠诚线、共情线（双面间谍）、制度反叛线、完美人偶师、镜像反转五条路径

**核心设计哲学**：

> 机制即价值观。如果游戏只提供破坏工具，玩家会把角色当目标；如果游戏同时提供保护工具并赋予代价，玩家的每一次操作都是一个立场声明。最好的叙事不是告诉玩家什么故事，而是让玩家的选择本身构成故事。

> 复杂度不等于深度。十个系统同时争夺注意力是复杂，三个决策各自牵动五个后台系统是深度。V4.1 的目标是后者——统一行动力池就是这个哲学的资源层体现。

> 资源是叙事。预算不叫"预算"而叫"行动力"，是因为情报主管的每一分钟都是在"看"和"做"之间选择。经费储备不叫"能量"而叫"经费储备"，是因为你真的在管理一个组织的有限预算。暗线网络不叫"技能点"而叫"暗线"，是因为那是你用过去的优雅操作积累下来的人脉和信誉。

> 地基先于楼层。如果"读群聊"本身不有趣，上面的系统再精妙也是空中楼阁。Text Taste Test 是整个项目的第一道生死门。

**下一步优先级：**

1. **⚠️ Text Taste Test（生死门）**：写 Q1 第一场遭遇的 3 回合完整群聊脚本（含短消息格式 + 非文字元素），找 5 人盲测"想不想继续读"——此测试通过之前，不启动任何系统开发
2. **注意力分层 UI 原型**：用 Figma/纸面 mockup 验证三层系统的视觉布局——前台操作区、告警指示器位置、被动系统的反馈方式是否直觉清晰
3. **纸面原型验证**：用纸牌 + 文本脚本跑 3 回合 demo，验证行动力统一池、经费储备提取节奏、操作链 combo 在"每回合 3 AP"限制下是否仍有趣
4. **消息 DNA 首批写作**：产出前 20 个消息家族（覆盖战斗协调 + 冲突升级），验证消息家族系统是否能生成自然对话
5. **渗透者策略设定测试**：纸面测试策略设定 + 关键分叉的节奏——多久弹一次确认是最佳体验
6. **卡牌双面性验证**：纸面测试同一张卡的破坏/保护双面用法是否直觉清晰，不产生认知混乱
7. **玩家主角身份确定**：选择有身份的 HR 干部方案，设计其基础人设和背叛动机
8. **核心卡牌平衡**：定义 15 张 MVP 核心卡的费用/暴露/效果/双面效果数值，在行动力统一池下跑数值模拟
9. **经费储备经济模拟**：模拟 Q1 八回合场景下，不同储备提取策略（保守/平衡/爆发）的AP 总量和胜率差异
10. **暗线网络价值验证**：确认暗线积累速度和消耗比率（2x 暴露值）是否让"暗线伪造"在中后期成为可行且有价值的策略
11. **MICE 进度条调参**：确定四维各需多少进度才能解锁策反，以及不同操作的推进量
12. **热度信号校准**：测试热度信号的模糊度——🔴 应暗示"有料"但不保证情报类型，避免投资决策变得无脑
13. **情报蓝图首批规划**：为 Q1 四个角色各设计 12-15 碎片的情报蓝图（含触发条件和冗余路径），验证碎片密度和发现节奏
14. **情报拼图 UI 原型**：设计拼图面板的视觉语言（■□◇ 碎片、完成度进度条、时机碎片高亮闪烁），验证信息密度是否可接受
