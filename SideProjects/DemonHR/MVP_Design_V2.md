# 📦 项目代号：《魔王的人力资源部》 (Demon HR: Hero Ruination) - V2.1 战役季度版

## MVP 极简描述

**游戏类型**：单机 RPG / 战役式 Deckbuilding / 轻 Roguelite / 多结局  
**视觉风格**：复古像素风 + 现代职场通讯软件 UI 包装  
**核心体验**：监听勇者小队通讯，利用性格弱点与组织手段瓦解团队，或将关键成员招安至魔王阵营

---

## 一、 产品定位 (Product Positioning)

本作不是纯《杀戮尖塔》式的高随机爬塔，也不是纯《欺诈之地》式的重剧情对话冒险，而是一个**固定战役结构 + 每季度局内构筑变化 + 角色跨季度持续存在**的单机战役型卡牌 RPG。

### 1.1 核心结构定义

- **整个游戏**：一个完整的 `Campaign`
- **每个季度**：一个固定目标小队的 `Run`
- **每个季度内**：一张小型地图，由多个遭遇、事件、休整点组成

### 1.2 核心卖点

- 玩家不是正面击败勇者，而是作为魔王军 HR 在幕后“做局”
- 勇者角色不会在一局后彻底消失，而会带着创伤、关系与履历进入后续季度
- 游戏存在多结局，玩家的立场会在“忠于魔王”与“共情勇者”之间逐步分化

---

## 二、 战役结构 (Campaign Structure)

### 2.1 总体思路

游戏由 `Q1-Q4` 或 `Q1-Q5` 组成。每个季度都是一个相对固定的 Run，玩家需要在 HR 资源耗尽前，瓦解当前季度负责的小队。

### 2.2 季度原型

- **Q1：初次出征勇者小队**
  - 作用：教学季度，建立“监听通讯 -> 干预 -> 结算”的基础循环
  - 特点：成员年轻、性格外露、关系简单，便于玩家快速理解系统
- **Q2：皇家骑士团**
  - 作用：强化季度间连续性
  - 特点：可能包含 Q1 存活成员，或与其有明确关系的新角色
- **Q3：精英远征队**
  - 作用：将小队矛盾升级为制度矛盾、荣誉矛盾与功绩矛盾
- **Q4：王国直属讨魔队 / 圣征军**
  - 作用：引入王国最强、最稳定、最难瓦解的官方精锐，把前几个季度积累的角色与后果推向高潮
- **Q5：终局季度（可选）**
  - 作用：根据前几个季度的存活者、招安者与重大事件，正式导向多结局

### 2.3 失败结构

- 若 **Q2 失败**，则**直接重打 Q2**，而不是回退到 Q1
- 设计原则：季度是一个清晰的章节边界，失败惩罚要存在，但不能过度摧毁叙事连续性
- 后续可讨论是否在高难度模式中加入“季度失败带轻度后果再重开”的变体

### 2.4 季度长度

- 每个季度为**一张小型地图**
- 小地图中包含：
  - 普通遭遇
  - 固定叙事事件点
  - 休整/牌组调整点
  - 季度 Boss 遭遇或终局事件

## 三、 核心视觉与信息流：小队通讯终端 (The Group Chat Interface)

玩家作为魔王军 HR，拥有勇者公会内部通讯的“后台管理员权限”。

### 3.1 双屏表达

- **左侧主屏（70%）**：勇者小队通讯群聊
  - 用于展示意图、矛盾、抱怨、误解、压力和关系变化
- **右侧副屏（30%）**：地牢战斗监控 / 执行层回放
  - 展示小型房间内的轻量自动战斗演出
  - 核心职责不是承载第二套主玩法，而是把左侧“做局”的后果以可观看、可理解、可复盘的方式表现出来
  - 采用“高速自动战斗 + 关键节点冻结/慢放 + 事故标签叠层”的表达方式，而不是纯文字日志或长时间看戏式自动战斗

### 3.2 信息展示原则

- 文本负责叙事乐趣与角色塑造
- 图标、标签和状态 Token 负责策略判断
- 玩家平时主要看摘要信息做决策，想看乐子时再阅读完整群聊
- 左侧是**主决策界面**，右侧是**后果验证与爽感界面**
- 玩家即使不长时间盯右侧，也应该能完成正确决策；右侧的价值在于增强因果感、事故感与角色记忆点

### 3.3 右侧副屏的设计原则

- **不是纯 MUD**：避免只用文字结算带来的时代感缺失和反馈不足
- **不是完整 Auto-Battler**：避免玩家进入长时间看戏和归因混乱
- **是执行层回放**：战斗只需在 2-5 秒内高密度展示关键动作与事故结果
- **强调关键节点**：如冲锋失误、治疗缺席、误伤、信任崩塌、招安机会出现时，自动冻结或慢放并弹出事件标签
- **摘要永远优先于表演**：表演必须服务于理解，而不是反过来让玩家靠看演出来猜系统

## 四、 单季度核心循环 (Quarter Run Loop)

每个季度的目标是：**在资源耗尽前，使目标小队失去继续推进任务的能力。**

### 4.1 单回合三阶段

#### 阶段一：情报监听 (Wiretapping Phase)

- 勇者在群聊中生成行动计划
- 系统提炼出关键意图：
  - 攻击目标
  - 防护承诺
  - 技能准备
  - 当前不满对象
- 暴露当前压力值、信任波动与潜在爆点

#### 阶段二：HR 干预 (Intervention Phase)

- 玩家消耗预算与手牌进行干预
- 干预分为两类：
  - **管理员权限**：如撤回消息、伪造指令、禁言、已读不回
  - **卡牌策略**：如挑拨、审计、奖励、加班、背锅、舆论污染

#### 阶段三：执行阶段 (Execution Phase)

- 勇者小队进入房间后，会进行一段**短时实时自动战斗演出**
- 演出的目标不是模拟完整战斗，而是展示“计划阶段被篡改后的后果”
- 该阶段的节奏是**快反馈**，单次演出应保持短促，并由系统自动高亮最关键的 2-3 个事故节点
- 结算内容包括：
  - HP 变化
  - 压力值变化
  - 信任关系变化
  - 团队凝聚力变化
  - 创伤、死亡、离队或倒戈事件

### 4.2 双阶段与非对称权限

本作采用“**计划阶段主构筑 + 执行阶段轻干预**”的双阶段结构。

#### 计划阶段：主决策层

- 玩家阅读群聊、识别人格弱点、打出主要卡牌与管理员操作
- 这里负责深度决策，例如：
  - 撤回承诺
  - 伪造命令
  - 关系挑拨
  - 舆论污染
  - 战前陷阱与预算布置

#### 执行阶段：即时干预层

- 玩家不再进行整手深度出牌，而只在关键时刻做少量临场操作
- 该阶段的卡牌与权限，不是第二套主玩法，而是对前期布局的“放大、补刀或触发”

### 4.3 左侧群聊作为主战场 (Social Combat Layer)

左侧通讯群聊不是纯叙事包装，也不是“看文本找关键词”的信息面板，而是本作真正的**主战场**。  
玩家在这里进行的不是传统伤害交换，而是一场围绕承诺、信任、压力、权威和信息传递的**社交战**。

#### 左侧主战场的核心博弈对象

- **承诺**
  - 例如“下回合我给你套盾”“你先扛，我第二拍开大”
  - 它们是群聊战斗中最重要的社交契约，也是右侧事故的主要因果来源
- **信任**
  - 决定角色是否相信队友、是否愿意照既定计划行动
- **压力**
  - 压力越高，角色越容易误读、失控、崩溃或进入可招安状态
- **权威**
  - 队长、圣骑士、资深成员的话语权不同，影响群聊中命令与背书的效力
- **注意力 / 信息传递**
  - 玩家可操纵谁看到了什么、谁的消息被淹没、哪句话被置顶、哪条私聊被泄露

#### 群聊中的真正“战斗单位”

左侧群聊的基础战斗单位，不应是单纯的角色数值，而应是**可操作的社交节点**，例如：

- 一个治疗承诺
- 一个冲锋命令
- 一次资源分配
- 一条抱怨
- 一次求救
- 一条私聊泄露

玩家优先干预的对象，是这些节点本身，而不是直接对角色做抽象数值削减。  
例如：

- 撤回承诺
- 伪造命令
- 延迟消息
- 放大抱怨
- 泄露私聊

节点被操纵后，再反向改变人物关系与执行结果。

#### 左侧战场的中间胜态

左侧群聊需要有自己的“战果”，而不只是给右侧附加一个模糊修正值。典型战果包括：

- **违约**：关键承诺未兑现
- **错位**：角色行动节奏不再统一
- **失声**：重要信息未能传达给关键成员
- **甩锅**：事故责任快速集中到某人身上
- **孤立**：角色被团队边缘化
- **动摇**：角色开始怀疑是否还应继续站在当前队伍一边
- **可招安**：角色不再把魔王阵营视为绝对敌人

这些中间胜态一旦成立，右侧战斗事故率与后续叙事张力都会显著上升。

#### 勇者方的群聊反制

若左侧只有玩家单方面操纵，而勇者只会被动挨打，则群聊不会形成真正的博弈。  
因此勇者队伍也必须具备基本的“修复能力”：

- **澄清**
  - 解释误会、否认伪造信息、恢复消息真实性
- **背书**
  - 通过队友作证来修复被破坏的信任
- **安抚**
  - 暂时压下争吵、降低压力、阻止内讧升级
- **重新分工**
  - 在计划被打乱后快速重建新的协作秩序

玩家的乐趣不只是“恶搞成功”，更是抢在勇者完成修复前，把矛盾推过临界点。

#### 左侧信息的三层结构

- **第一层：聊天流**
  - 负责角色魅力、代入感和戏剧性
- **第二层：提炼后的战术信息**
  - 负责把关键信息抽取成一眼可读的策略节点
- **第三层：关系 / 状态板**
  - 负责长期变量，如信任、压力、权威、创伤和可招安性

设计原则是：**文本负责好看，结构化信息负责能玩。**

#### MVP 的最小闭环

MVP 阶段左侧群聊建议只先做以下六类核心变量：

- 承诺
- 信任
- 压力
- 权威
- 消息状态（正常 / 延迟 / 撤回 / 伪造 / 泄露）
- 人格触发（多疑、傲慢、贪婪、暴躁、自卑、忠诚）

围绕这些变量，优先实现五类核心操作：

- 撤回
- 伪造
- 延迟
- 放大
- 泄露

只要这套闭环成立，左侧群聊就不再是“有趣包装”，而会成为真正撑起本作辨识度的主系统。

### 4.4 三回合 Demo 推演（体验验证样例）

以下推演用于验证：左侧群聊战斗是否真的能形成“读局 -> 布局 -> 执行 -> 余波 -> 再布局”的连续乐趣，而不是停留在概念层。

#### 场景设定

- 季度：`Q1 初次出征勇者小队`
- 地图节点：`补给仓库走廊`
- 小队构成：
  - **队长 / 战士**：`傲慢 + 暴躁`
  - **牧师**：`忠诚 + 自卑`
  - **法师**：`多疑 + 抢功`
  - **盗贼**：`贪婪 + 滑头`
- 玩家当前资源：
  - 预算：`3`
  - 计划阶段手牌：`消息撤回`、`公开引用`、`已读不回`、`人事背锅`
  - 执行阶段反应卡：`增援呼叫`

#### 回合 1：埋下第一道裂痕

**群聊情报：**

- 战士：“前面仓库房肯定有埋伏，我先进，牧师给我挂治疗，法师第二拍清后排。”
- 牧师：“收到，我会盯着你的血线。”
- 法师：“别又是我收拾残局就行。”
- 盗贼：“要是有宝箱先别乱开，上次你们分配太离谱了。”

**系统提炼出的关键节点：**

- `承诺：牧师 -> 战士 [治疗支援]`
- `命令：战士 -> 全队 [战士先手开怪]`
- `抱怨：法师 -> 队伍 [不满被当收尾工具]`
- `抱怨：盗贼 -> 队伍 [不满战利品分配]`

**玩家决策点：**

此时玩家有至少两种合理切口：

- 破坏治疗链，制造前排事故
- 放大法师抱怨，埋下指挥冲突

**玩家操作：**

1. 对 `承诺：牧师 -> 战士 [治疗支援]` 使用 `消息撤回`
2. 对 `抱怨：法师 -> 队伍 [不满被当收尾工具]` 使用 `公开引用`

**策略意图：**

- 让战士误以为自己仍有稳定后援
- 让法师的不满从“背景噪音”升级为“公开站队”

**执行阶段：**

- 战士冲锋进入前排
- 牧师节奏被打乱，没有形成稳定治疗链
- 法师开始蓄力，但团队氛围已明显变差
- 战士血量掉得比预期更快，触发 `Reaction Window`

**反应窗口：**

- 可用选项：`增援呼叫`

**玩家操作：**

- 打出 `增援呼叫`，在后排刷出额外投矛怪

**回合结果：**

- 战士濒危但未死
- 牧师没能及时奶上
- 法师的抱怨被现实验证

**战后余波：**

- 战士：“牧师你刚才在干嘛？我差点死了！”
- 牧师：“我有抬手啊！后排突然多了一只怪，我没来得及……”
- 法师：“我就说别每次都让我收残局。”

**状态变化：**

- 战士 -> 牧师 信任 `-20`
- 战士 -> 法师 信任 `-10`
- 牧师 压力 `+12`
- 法师 压力 `+8`
- 团队凝聚力 `-18`
- 新增中间胜态：`违约`、`甩锅`

---

#### 回合 2：把裂痕从事故变成站队

**群聊情报：**

上回合事故后，队伍已不再处于稳定合作状态。

- 战士：“这次别废话，牧师先跟紧我，法师你别再划水。”
- 牧师：“我会尽量跟上……”
- 法师：“你冲那么快，谁跟得上？”
- 盗贼：“后面有个锁箱，我先去看一眼。”

**系统提炼出的关键节点：**

- `命令：战士 -> 牧师 [优先跟随前排]`
- `命令：战士 -> 法师 [服从节奏]`
- `抱怨：法师 -> 战士 [质疑冲锋节奏]`
- `请求：盗贼 -> 队伍 [默许脱队搜箱]`

**玩家决策点：**

这一回合重点已经不是单纯制造伤害，而是决定把队伍推向哪种崩坏：

- 继续打前后排配合，逼牧师崩溃
- 或放大法师与队长的权威冲突
- 或诱导盗贼脱队，让团队出现“纪律问题”

**玩家操作：**

1. 对 `抱怨：法师 -> 战士 [质疑冲锋节奏]` 使用 `公开引用`
2. 对 `请求：盗贼 -> 队伍 [默许脱队搜箱]` 使用 `已读不回`

**策略意图：**

- 让队长明确感受到法师不服从
- 让盗贼脱队行为得不到及时约束，制造队伍错位

**执行阶段：**

- 战士继续冲锋
- 牧师被迫前压，站位危险
- 法师消极执行，不愿完全配合队长节奏
- 盗贼偏离主队去摸箱子，导致后排保护缺口

**战斗结果：**

- 后排被侧翼小怪切入
- 牧师承受额外压力
- 法师清场效率下降
- 队伍出现明显站位错位

**战后余波：**

- 战士：“你们到底有没有一个人听指挥？”
- 法师：“你那不叫指挥，叫送死。”
- 盗贼：“我就离开半分钟，至于吗？”
- 牧师：“别吵了……再这样下去下一间房肯定出事。”

**状态变化：**

- 战士 权威 `-15`
- 法师 -> 战士 信任 `-15`
- 牧师 压力 `+10`
- 盗贼 团队忠诚 `-10`
- 团队凝聚力 `-15`
- 新增中间胜态：`错位`、`动摇`

---

#### 回合 3：把团队推到临界点

**群聊情报：**

前两回合后，队伍表面还在运作，但内部已接近失控。

- 战士：“最后一间，谁再乱来我就不管了。”
- 牧师：“要不我们先休整一下……”
- 法师：“现在知道怕了？”
- 盗贼：“前面那扇门后面可能有大货，我先声明，谁都别抢。”

**系统提炼出的关键节点：**

- `威胁：战士 -> 全队 [停止保护]`
- `请求：牧师 -> 全队 [暂停推进 / 休整]`
- `抱怨：法师 -> 战士 [嘲讽权威]`
- `资源冲突：盗贼 -> 全队 [预占战利品归属]`

**玩家决策点：**

这一回合不再只是制造小事故，而是决定队伍会走向哪种终局：

- 让战士彻底甩锅，触发公开决裂
- 让牧师崩溃，导致支援系统失效
- 让盗贼引爆资源分配争议，触发公开内讧

**玩家操作：**

1. 对 `请求：牧师 -> 全队 [暂停推进 / 休整]` 使用 `已读不回`
2. 对 `资源冲突：盗贼 -> 全队 [预占战利品归属]` 使用 `公开引用`
3. 对 `威胁：战士 -> 全队 [停止保护]` 使用 `人事背锅`，把潜在事故责任提前引导到牧师与法师身上

**策略意图：**

- 阻止队伍通过休整进行自我修复
- 在进入最终房间前先把资源争议公开化
- 让战士在事故发生后更容易迅速甩锅

**执行阶段：**

- 队伍带着未解决的矛盾进入房间
- 战士因高压与低信任状态，保护欲下降
- 牧师因高压力与低自信，治疗反应迟疑
- 法师因被持续质疑，选择优先自保
- 盗贼因贪婪触发，走位偏向战利品区域

**战斗结果：**

- 前排与后排完全脱节
- 一名成员濒死，另一名成员拒绝第一时间救援
- 战斗虽然勉强结束，但队伍的“继续冒险共识”彻底瓦解

**战后余波：**

- 战士：“我受够了，这队谁爱带谁带。”
- 法师：“那就别带了。”
- 盗贼：“战利品我拿走，反正你们也配不上。”
- 牧师：“……我们真的还要继续吗？”

**最终状态：**

- 团队凝聚力跌破阈值
- 队伍进入 `解散边缘` 状态
- 牧师进入 `可招安` 边缘状态
- 战士与法师进入 `公开敌对`

---

#### 这个 Demo 想验证什么

这三回合推演不是为了证明当前设计已经成立，而是为了验证以下几个问题：

1. 玩家每回合是否真的有“选裂口”的乐趣，而不是只有唯一最优解
2. 左侧群聊操作是否能自然累积成右侧事故，而不是依赖随机解释
3. 战后余波是否足够强，能把数值变化放大成角色关系变化
4. 玩家是否会开始对具体角色形成偏心、厌恶或招安欲望
5. `违约 -> 甩锅 -> 错位 -> 动摇 -> 解散/招安` 这条链路是否足够自然

若这条体验链成立，则说明左侧群聊主战场具备继续细化的价值；若不成立，则应继续收窄节点类型、状态量与玩家操作维度。

### 4.5 Reaction Window（反应窗口）

执行阶段允许玩家继续干预，但必须通过**事件窗口**触发，而不是全程任意暂停自由出牌。

#### 触发原则

- 只在关键节点出现
- 每次提供少量、高价值、直观的反应选择
- 目的是放大戏剧性，而不是重做一遍主策略判断

#### 典型触发时机

- 战士即将冲锋
- 牧师准备治疗或支援
- 队长下达关键命令
- 某成员血量低于阈值
- 两名成员的信任跌破阈值
- 角色进入可招安状态

#### 反应窗口中的可用干预

- **环境卡**
  - 例如：增援呼叫、地板塌陷、封门、照明故障
- **反应卡**
  - 例如：断线、误报、恐吓广播、人事冻结
- **条件处决 / 招安卡**
  - 例如：火上浇油、补刀舆论、招安话术

#### 反应窗口中的设计禁区

- 不应要求玩家阅读大量文本
- 不应让玩家重新做完整策略规划
- 不应把执行阶段扩张成第二套卡牌主循环

### 4.6 单局体验目标

理想体验为：

1. 玩家在左侧群聊中完成布局与操盘
2. 右侧战斗快速证明这些布局确实造成了物理和情绪后果
3. 玩家在少量关键时刻插手补刀，制造更强的事故、内讧或招安时机
4. 结算后回到群聊余波，让角色继续用对话放大后果与情感记忆
### 4.7 胜利条件 (KPI)

- **团灭 (Wipeout)**：全员 HP 归零，传统胜利，评价较低
- **解散 (Disband)**：团队凝聚力归零，成员退群或拒绝继续冒险，评价较高
- **招安 (Recruitment)**：关键成员被说服、逼迫或诱导加入魔王阵营，评价最高且会改写后续季度结构

## 五、 招安机制 (Recruitment Route)

### 5.1 招安是什么意思

“招安”不是单纯的隐藏彩蛋，也不是“把敌人变成卡牌”这么简单。  
它是本作的一条**正式战略路线**：玩家不是只追求杀死或解散勇者，而是通过连续干预，让某个角色逐渐失去对王国、公会或现队友的信任，最终接受魔王一方提供的位置、保护或价值认同。

### 5.2 招安的成立条件

一个角色进入可招安状态，通常需要同时满足几类条件：

- **组织失望**：对队长、公会、王国或同伴失去信任
- **情绪崩溃**：压力过高、长期被误解、被当作消耗品
- **利益诱因**：看到了魔王阵营能提供的利益、尊重或生存空间
- **人格契合**：不同人格会响应不同话术

### 5.3 招安的几种路径

- **利益型招安**
  - 例如贪婪型角色接受魔王军提供的资源、职位或财富
- **情感型招安**
  - 例如长期被队伍忽视的角色，转而相信“至少魔王军看见了自己”
- **理念型招安**
  - 例如对王国体制失望的角色，认为勇者公会和魔王城本质上没有差别，最终选择站到新的秩序一边

### 5.4 招安的系统意义

- 招安角色可在后续季度中：
  - 以 NPC 形式回归
  - 提供情报、特殊卡牌或组织权限
  - 改写某些事件的结果
- 招安不是额外奖励，而是会影响战役分支与结局的核心系统

## 六、 角色持续化 (Persistent Characters)

### 6.1 持续化深度

本作中的角色持续化不是只保留一个名字，而是保留以下内容：

- 名字
- 职业与性格标签
- 完整人格倾向
- 关键关系
- 创伤记录
- 履历标签

### 6.2 持续化结果类型

一个角色在季度结束后，可能进入以下状态：

- 战死
- 重伤退役
- 存活并转入新小队
- 被招安至魔王阵营
- 精神崩溃退出冒险体系

### 6.3 持续化的叙事作用

- 让 Q1 的情感积累在 Q2/Q3 有回报
- 让玩家对“自己造成的后果”形成长期记忆
- 让游戏不只是重复开新局，而是真正推进一段组织史与人物命运

## 七、 玩家如何变强 (Player Growth)

### 7.1 卡组构筑成长

- 每个季度推进过程中，玩家会获得：
  - 新卡
  - 删牌机会
  - 升级牌机会
- 构筑的重点不是单卡强度，而是“针对某类人格和关系的组合打法”

### 7.2 权限成长

玩家的成长不只体现在手牌上，还体现在 HR 权限的提升上：

- 基础监听
- 撤回消息
- 伪造指令
- 禁言
- 审计履历
- 篡改绩效
- 错发补给 / 错发命令

### 7.3 情报成长

- 玩家会积累老兵档案、创伤记录与关系史
- 同一角色再次出现时，玩家并不是第一次面对他，而是“知道如何击穿他”

## 八、 核心机制模块 (Key Systems)

### 8.1 性格标签弱点系统

角色由 `[职业] + [性格A] + [性格B] + [关系状态]` 构成。  
真正决定玩法的不是血量，而是这些标签如何与通讯事件、组织手段和突发事故产生连锁反应。

### 8.2 团队凝聚力系统

- 凝聚力代表小队是否还能作为一个“组织”继续运作
- 它会受到以下因素影响：
  - 误解
  - 背锅
  - 资源分配不公
  - 指令冲突
  - 队友死亡
  - 长期低信任关系

### 8.3 动态声望与公会应对

- 玩家长期偏好某种流派时，王国公会会调整派遣队伍结构
- 不是给角色硬免疫，而是从源头改变来者是谁
- 这样迫使玩家季度间不断重构卡组

### 8.4 固定事件节点

每个季度内包含固定触发点，例如：

- 某成员可能战死
- 某成员可能升职或转队
- 某起事故会成为下季度的舆论事件

设计原则是：**节点固定，结果可变。**

## 九、 多结局结构 (Endings)

### 9.1 忠诚线：魔王提拔结局

- 玩家始终忠于魔王利益
- 瓦解或消灭所有小队
- 最终获得魔王赏识，晋升为核心干部

### 9.2 共情线：放走勇者结局

- 玩家与某位关键角色建立情感连接
- 最终帮助其带队走向胜利，或至少成功逃离组织命运

### 9.3 第三路线：待讨论

- 可考虑做成“既不忠于魔王，也不完全帮助勇者”的制度反叛结局
- 当前先保留为可选方向，不在 MVP 中强行锁死

## 十、 当前待讨论模块 (Open Questions)

### 10.1 玩家主角身份

当前已确认：玩家是魔王军 HR 体系中的核心操作员。  
当前未确认：玩家是否是

- 无脸管理员
- 有具体身份与履历的 HR 干部
- 可成长、可被结局反转定义的主角角色

这会直接影响：

- 叙事代入方式
- 结局情感力度
- 玩家是否能成为剧情角色的一部分

### 10.2 MVP 建议收束

若以最小可玩版本推进，建议：

- 先做 `Q1-Q3`
- 先做 `6-8` 个可持续角色
- 先做三条策略路线：
  - 离间
  - 压榨
  - 招安

## 十一、 设计结论 (Current Direction)

《魔王的人力资源部》的 V2.1 方向已经明确为：

- 一个**固定季度结构**的战役型 Deckbuilding RPG
- 每个季度是一张**小型地图**
- 失败时**重打当前季度**
- 角色会以**名字、标签、人格、关系、创伤**持续存在
- “招安”是与“团灭”“解散”并列的**正式主策略路线**

下一步最值得继续讨论的是：**玩家主角身份**与**招安路线的具体操作流程**。
