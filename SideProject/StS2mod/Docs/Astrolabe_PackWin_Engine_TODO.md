## 一、目标定义

> 这里的“包赢”不是数学意义上的 100% 必胜，而是对绝大多数正常局面都能给出高质量、低失误、可解释、可执行的建议，让玩家主观体验接近“被顶级教练全程带飞”。

### 1.1 体验目标

- [ ] 玩家在每个关键决策点都能立刻看到**明确推荐 + 简短理由 + 风险提示**。
- [ ] 玩家不需要自己在地图、商店、篝火、事件、战斗之间重新拼上下文。
- [ ] 推荐像“副驾在替你盯全局”，而不是“每个界面各说各话”。
- [ ] 高端局不只会说正确的废话，而是能指出**这局真正的胜利路径**。

### 1.2 客观指标

- [ ] 建立固定种子测试集（按职业 / Act / 稀有局面分层）。
- [ ] 在固定测试集上记录基线胜率、到达 Act 3 比例、Boss 击杀率、平均剩余 HP、路线偏差率。
- [ ] 在核心职业上实现：
  - [ ] 新手 pilot 的胜率显著提升。
  - [ ] 中级 pilot 的平均局质量显著提升。
  - [ ] 高级 pilot 仍能从建议中获得稳定增益。
- [ ] 非战斗建议平均响应时间控制在可接受范围内。
- [ ] 战斗重算在玩家出牌节奏内完成，不打断操作流。
- [ ] 所有建议都能附带可解释理由，而不是纯黑盒分数。
- [ ] 建立“建议是否命中正确方向”的回放复盘体系。

### 1.3 当前状态判断

- [x] 已有可运行主链：`Hook -> RunStateReader -> BuildPathManager -> AdvisorEngine -> HUD/UI`
- [x] 已有战斗建议支线：`CombatHook -> CombatAdvisor -> OverlayHUD`
- [x] 已有部分跨界面上下文桥接：`CampfireHook -> DeckUpgradeHook`
- [ ] 还没有整局统一策略脑。
- [ ] 还没有真实地图搜索。
- [ ] 还没有完整的商店 / 事件 / 宝箱 / BossRelic 决策闭环。
- [ ] 还没有强鲁棒性的评测、回放、自动调参体系。
- [ ] 还远未达到“包赢感”所需的稳定度。

## 二、总体推进原则

### 2.1 绝对优先级

1. **先做正确的决策链路，再做花哨 UI。**
2. **先做可评测，再做更复杂的 heuristics。**
3. **先打通整局闭环，再优化单点聪明度。**
4. **先减少大错，再追求极限最优。**
5. **任何“包赢”能力都必须能被复盘验证。**

### 2.2 工程原则

- [ ] 统一“建议输入”和“建议输出”模型，避免每个 Hook 自己拼数据。
- [ ] 所有重要建议必须带 `confidence / risk / why / alternative`。
- [ ] 所有重要建议必须能落日志，供回放和离线评估使用。
- [ ] 所有复杂策略都要有 fallback，超时或数据缺失时不能胡乱强推。
- [ ] 不过早追求全自动，优先做“高质量副驾”。

## 九、终局目标复述

> Astrolabe 的终局形态，不是一个“偶尔说对几句话”的提示插件，
> 而是一个**知道你这局要赢什么、知道你现在最怕什么、知道下一步该去哪、知道这回合该怎么打、还能把理由讲清楚的整局战略引擎**。

这份 TODO 的全部任务，都是围绕这一个目标服务：

**让玩家在绝大多数局面里，都感觉自己被一个真正会赢的人带着走。**
