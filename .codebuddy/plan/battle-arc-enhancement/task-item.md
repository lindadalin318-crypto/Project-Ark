# 实施计划：战斗弧线增强 + 物理模拟完善

- [ ] 1. 修改陀螺初始位置为对角出发
   - 在 `startBattleLoop()` 中将玩家陀螺初始位置设为左下角（`ARENA.radius * 0.75` 方向），敌人陀螺设为右上角对角位置
   - 保留原有 `launchAngle` / `launchPower` 初始速度计算逻辑，仅改变起始坐标
   - _需求：1.1、1.2、1.3_

- [ ] 2. 实现碰撞冷却帧机制
   - 在陀螺状态对象上新增 `_collisionCooldown` 计数器字段，初始值为 0
   - 在 `physicsStep()` 每帧将 `_collisionCooldown` 递减 1（最小归零）
   - 在 `checkTopCollision()` 中：冷却期内跳过 RPM 伤害计算，但保留速度反弹和物理分离逻辑
   - _需求：2.1、2.2、2.3、2.4_

- [ ] 3. 实现濒死保护期机制
   - 在陀螺状态对象上新增 `_dyingProtection`、`_dyingProtectionUsed`、`_dyingProtectionStartTime` 字段
   - 在 `physicsStep()` 中每帧检测 `rpm / maxRpm < 0.15`，若未使用过保护则激活保护期
   - 保护期内：RPM 自然衰减乘以 `0.3`，碰撞伤害乘以 `0.4`
   - 保护期满 4 秒后自动结束，恢复正常衰减和伤害，不再重新触发
   - _需求：3.1、3.2、3.3、3.4、3.5、3.6_

- [ ] 4. 完善 Magnus 效应——补充同向自旋切向偏转
   - 在 `checkTopCollision()` 的 Magnus 切向冲量计算段，识别两陀螺自旋方向是否相同
   - 同向自旋：施加方向取反、幅度为 `magnusTangentRatio * 0.6` 的切向冲量
   - 对向自旋：保持现有全量 `magnusTangentRatio` 逻辑不变
   - RPM < 20% 时的衰减系数（30%）对同向自旋同样生效
   - _需求：4.1、4.2、4.3、4.4_

- [ ] 5. 更新 BattleLog 记录新机制事件
   - 碰撞冷却期内触发碰撞时输出 `[冷却中-跳过伤害]` 及剩余冷却帧数（需求 2.5）
   - 濒死保护期激活/结束时输出陀螺身份、当前 RPM 百分比、保护期时长（需求 3.7、5.1）
   - 战斗结束摘要中追加：总碰撞次数（含冷却跳过）、濒死保护触发次数、首次碰撞时间戳（需求 5.3）
   - _需求：2.5、3.7、5.1、5.2、5.3_
