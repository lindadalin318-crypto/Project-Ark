# 需求文档：LevelDesigner.html 效率升级

## 引言

LevelDesigner.html 是 Project Ark 的浏览器端关卡可视化编辑工具。当前工具已具备基础的房间拖拽、连接、元素配置和 JSON 导出功能。

本次升级的核心诉求是：**方便、快捷、尽量一步到位，减少整个制作关卡流程中琐碎的步骤**。

参考文档：`Docs/GDD/示巴星 (鸣钟者)/关卡心流与节奏控制-2.csv`，该 CSV 包含 27 个关卡节点，每个节点有 Zone ID（如 Z1a、Z2d）、ACT 分组、张力值（1-10）、房间类型等字段，这些信息目前需要在 Unity 中手动补充，应在工具中直接支持。

---

## 需求

### 需求 1：房间属性扩展——补充关键字段

**用户故事：** 作为一名关卡设计师，我希望在 LevelDesigner.html 中直接填写 Zone ID、ACT 分组、张力值、叙事乐章名称等字段，以便导出的 JSON 能被 ScaffoldToSceneGenerator.cs 直接使用，无需在 Unity 中手动补充。

#### 验收标准

1. WHEN 用户选中一个房间 THEN 属性面板 SHALL 显示以下新增字段：
   - **Zone ID**（文本输入，如 `Z1a`、`Z2d`）
   - **ACT 分组**（下拉选择：ACT1 / ACT2 / ACT3 / ACT4 / ACT5）
   - **张力值**（数字滑块，范围 1-10，整数）
   - **叙事乐章**（文本输入，如 `序曲：坠落`）
   - **时长估算**（文本输入，如 `0:00-5:00`）
2. WHEN 用户修改上述字段 THEN 系统 SHALL 实时更新房间数据并反映在 JSON 实时预览中
3. WHEN 导出 JSON THEN 上述字段 SHALL 包含在每个 room 对象中（字段名：`zoneId`、`act`、`tension`、`beatName`、`timeRange`）
4. IF 字段为空 THEN 导出时 SHALL 省略该字段（不输出 null 或空字符串）

---

### 需求 2：房间类型扩展——补充示巴星专用类型

**用户故事：** 作为一名关卡设计师，我希望房间类型选项能覆盖示巴星的所有房间类型，以便在工具中一次性完成分类，不需要在 Unity 中再手动修改。

#### 验收标准

1. WHEN 用户从左侧面板拖拽房间 THEN 可选类型 SHALL 包含：
   - `safe`（安全区，绿色）
   - `normal`（普通，橙色）
   - `arena`（竞技场，红色）
   - `boss`（Boss，紫色）
   - `narrative`（叙事/过场，蓝紫色）
   - `puzzle`（解谜，青色）
   - `corridor`（走廊/过渡，灰色）
2. WHEN 用户在属性面板修改房间类型 THEN 房间的视觉颜色 SHALL 立即更新为对应颜色
3. WHEN 导出 JSON THEN 房间类型 SHALL 使用英文 key（`narrative` / `puzzle` / `corridor`）

---

### 需求 3：元素类型扩展——补充示巴星专用元素

**用户故事：** 作为一名关卡设计师，我希望元素类型能覆盖示巴星的特殊机制元素，以便在工具中直接标注，减少 Unity 中手动添加组件的步骤。

#### 验收标准

1. WHEN 用户从左侧面板拖拽元素 THEN 可选元素类型 SHALL 新增：
   - `tidal_door`（潮汐门，蓝绿色，仅涨潮期开启）
   - `resonance_crystal`（共鸣水晶，金色，降热/解谜用）
   - `lore_audio`（Lore 音频日志，白色）
   - `black_water`（黑水区域，深黑色）
2. WHEN 导出 JSON THEN 新增元素类型 SHALL 使用对应英文 key

---

### 需求 4：心流张力可视化——内置张力曲线图

**用户故事：** 作为一名关卡设计师，我希望在工具中直接看到当前关卡设计的张力曲线，以便实时对照 CSV 中的心流设计，不需要手动核对数据。

#### 验收标准

1. WHEN 画布中存在至少 2 个有张力值的房间 THEN 右侧面板 SHALL 在 ASCII 预览区域上方显示一个迷你张力折线图
2. WHEN 折线图渲染 THEN 系统 SHALL 按照房间的 ACT 分组和 Zone ID 排序（Z1a→Z1b→...→Z5c）
3. WHEN 折线图渲染 THEN 每个节点 SHALL 显示 Zone ID 标签和张力数值
4. WHEN 用户修改某个房间的张力值 THEN 折线图 SHALL 实时更新
5. WHEN 折线图中某段张力变化超过 4（如从 3 跳到 8）THEN 该段连线 SHALL 高亮显示（提示设计师注意节奏跳跃）

---

### 需求 5：快捷键支持——高频操作提速

**用户故事：** 作为一名关卡设计师，我希望常用操作有快捷键支持，以便减少鼠标点击次数，提升设计效率。

#### 验收标准

1. WHEN 用户按 `Ctrl+Z` THEN 系统 SHALL 撤销上一步操作（支持：添加房间、删除房间、移动房间、添加连接、删除连接、添加元素、删除元素）
2. WHEN 用户按 `Ctrl+D` 且有房间被选中 THEN 系统 SHALL 复制该房间（包含其所有属性和元素，但不复制连接关系）
3. WHEN 用户按 `Ctrl+S` THEN 系统 SHALL 执行保存到本地存储操作（等同于点击"保存"按钮）
4. WHEN 用户按 `Escape` THEN 系统 SHALL 取消当前连接操作（如果正在拖拽连接线）
5. WHEN 快捷键触发时焦点在输入框内 THEN 系统 SHALL 不触发快捷键（避免干扰文字输入）

---

### 需求 6：ACT 分组框——视觉分组管理

**用户故事：** 作为一名关卡设计师，我希望画布上的房间能按 ACT 分组显示分组框，以便在 27 个节点平铺时快速识别各 ACT 的边界，不会混乱。

#### 验收标准

1. WHEN 画布中存在设置了 ACT 分组的房间 THEN 系统 SHALL 在这些房间背后渲染一个半透明的分组框（每个 ACT 一个颜色）
2. WHEN 分组框渲染 THEN 框的边界 SHALL 自动包围该 ACT 下所有房间（带 20px padding）
3. WHEN 分组框渲染 THEN 框的左上角 SHALL 显示 ACT 标签（如 `ACT 1`）
4. WHEN 用户移动房间 THEN 分组框 SHALL 实时更新边界
5. WHEN 某个 ACT 下只有 1 个房间 THEN 分组框 SHALL 正常显示（不隐藏）

---

### 需求 7：连接线语义类型——减少 Unity 端配置

**用户故事：** 作为一名关卡设计师，我希望连接线能标注语义类型（普通通道/潮汐门/锁门/单向），以便导出的 JSON 包含足够信息，减少在 Unity 中手动配置 Door 组件的步骤。

#### 验收标准

1. WHEN 用户选中一条连接线 THEN 属性面板 SHALL 显示连接类型下拉选择：
   - `normal`（普通通道，红色线，默认）
   - `tidal`（潮汐门，蓝绿色线，仅涨潮期开启）
   - `locked`（锁门，黄色线，需要钥匙）
   - `one_way`（单向，橙色线）
2. WHEN 用户修改连接类型 THEN 连接线颜色 SHALL 立即更新为对应颜色
3. WHEN 导出 JSON THEN 每条连接 SHALL 包含 `connectionType` 字段

## 边界情况

- **撤销历史上限**：撤销历史最多保留 50 步，超出后丢弃最旧的记录
- **分组框性能**：当房间数量超过 50 个时，分组框更新应节流（16ms 间隔）
- **张力曲线排序**：Zone ID 排序规则为字母数字混合排序（Z1a < Z1b < Z2a < Z10a）
- **复制房间 ID 冲突**：复制房间时，新房间 ID 应自动添加 `_copy` 后缀并确保唯一性
- **向后兼容**：导入旧格式 JSON（不含新字段）时应正常工作，新字段使用默认值
