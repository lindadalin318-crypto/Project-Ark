# 实施计划：示巴星 ACT1+ACT2 关卡布局重构

> 目标文件：`Docs/LevelDesigns/Sheba_ACT1_ACT2.json`
> 格式参考：`Tools/LevelDesigner.html` 的 `getExportData()` 输出格式
> 心流参考：`Docs/GDD/示巴星 (鸣钟者)/关卡心流与节奏控制-2.csv`

---

- [ ] 1. 设计 ACT1 Z1a 坠机点区域（张力 4）— 起点 + 第一个分支
   - 重写 `rooms` 中 Z1a 相关房间（3~4 个房间），设定为起点区域
   - 位置布局：Z1a 作为地图左侧起点（floor=0），向东和向南提供两条初始前进路径（需求 1.1）
   - 元素配置：spawn 点 + checkpoint + 基础 enemy（需求 4.1、4.2、4.3）
   - 所有 Z1a 内部连接使用双向 connections（A→B 和 B→A）（需求 5.7）
   - _需求：1.1, 1.4, 4.1, 4.2, 5.4, 6.1_

- [ ] 2. 设计 ACT1 Z1b 寂静雪原区域（张力 2）— 探索 + 隐藏地穴
   - 创建 Z1b 房间群（3~4 个房间），包含 floor=0 主层和 floor=-1 隐藏地穴
   - 位置布局：从 Z1a 东侧延伸，地穴在下方（垂直层级，需求 2.1）
   - 元素配置：safe 类型 + checkpoint + npc + chest（需求 4.2、4.5）
   - 地穴用 north/south 方向连接表示垂直过渡（需求 2.4）
   - 这是低张力探索区，空间应更开阔，周围有可选探索分支（需求 6.4）
   - _需求：1.4, 2.1, 2.4, 2.5, 4.2, 4.5, 6.1, 6.4_

- [ ] 3. 设计 ACT1 Z1c 觉醒战场区域（张力 7）— 高张力竞技场
   - 创建 Z1c 房间群（3~4 个房间），包含主竞技场 + 侧路 + 上层观景台（floor=1）
   - 位置布局：从 Z1b 东侧进入，侧路向南分支，观景台在上方（需求 2.1）
   - 元素配置：arena 类型 + 大量 enemy + chest（需求 4.1、4.3）
   - Z1c 侧路应与 Z1b 区域形成**第一个环路**（需求 1.3）
   - 高张力区域空间紧凑封闭（需求 6.3）
   - _需求：1.3, 1.4, 2.1, 4.1, 4.3, 6.1, 6.3_

- [ ] 4. 设计 ACT1 Z1d 音叉林区域（张力 4）— 垂直探索 + 安全区
   - 创建 Z1d 房间群（3~4 个房间），包含 floor=0 中层和 floor=1 上层
   - 位置布局：从 Z1c 东侧进入，音叉林有上下层探索（需求 2.2）
   - 包含「花园·安全区」作为关键 checkpoint（需求 4.2）
   - 设计从 Z1d 回到 Z1b 的**单向捷径连接**（跌落/一次性门），形成第一条 shortcut（需求 3.1）
   - 单向连接在 JSON 中只有一条 connection A→B（需求 5.8）
   - _需求：1.3, 2.2, 2.4, 3.1, 4.2, 5.8, 6.1_

- [ ] 5. 设计 ACT1 Z1e+Z1f 区域（张力 3→6）— 反思 + 首棱镜 + ACT1 出口
   - 创建 Z1e 房间群（2~3 个房间）：认知转折 + 道德选择房间
   - 创建 Z1f 房间群（3~4 个房间）：棱镜竞技场 + 缕拉目击点 + ACT1 出口
   - Z1e 从 Z1a 区域南向路径汇入（第二条路径的汇聚点，需求 1.2）
   - Z1f 棱镜竞技场作为 ACT1 高潮战斗（需求 6.1）
   - ACT1 出口 door 连接到 ACT2 入口（需求 4.4）
   - 确保 Z1e 低张力区有可选支线分支（需求 6.4）
   - _需求：1.2, 1.5, 3.4, 4.1, 4.4, 6.1, 6.4_

- [ ] 6. 设计 ACT2 Z2a 管风琴走廊区域（张力 5）— 峡谷降临 + 垂直层级
   - 创建 Z2a 房间群（3~4 个房间），包含 floor=0 和 floor=-1（需求 2.3）
   - 位置布局：从 ACT1 出口向南/向下进入峡谷（暗示垂直下降）
   - 包含上层平台（floor=0）和下层走廊（floor=-1）（需求 2.3、2.5）
   - 元素配置：normal 类型 + enemy + npc + checkpoint（需求 4.1）
   - 缕拉远程援助房间为 safe 类型（需求 4.2）
   - _需求：2.3, 2.4, 2.5, 4.1, 4.2, 6.2_

- [ ] 7. 设计 ACT2 Z2b 涟漪获取区域（张力 3）— 解谜 + NPC 面对面
   - 创建 Z2b 房间群（3~4 个房间）：谜题房 + 涟漪奖励 + 缕拉面对面
   - 从 Z2a 下层进入，布局向东南延伸
   - 设计一条从 Z2b 隐藏区域到 Z2c 的**单向跌落连接**（需求 3.3）
   - 涟漪获取后暗示有新的可探索路径（需求 3.2）
   - 低张力区域应有可选支线探索（需求 3.4、6.4）
   - _需求：3.2, 3.3, 3.4, 4.2, 4.5, 5.8, 6.2_

- [ ] 8. 设计 ACT2 Z2c 叹息深谷区域（张力 5）— 涟漪实战 + 世界时钟预兆
   - 创建 Z2c 房间群（4~5 个房间）：环境解谜 + 战斗验证 + 回音路 + 频率标记
   - 布局形成网状：回音路与战斗验证房间形成**第三个环路**（需求 1.3）
   - 世界时钟预兆房间和频率标记点双路径汇合到 Z2d 前室（需求 1.2）
   - 包含隐形平台相关的门连接（需求 4.4）
   - _需求：1.2, 1.3, 4.1, 4.4, 6.2_

- [ ] 9. 设计 ACT2 Z2d Boss 战区域（张力 9）— Boss 竞技场 + 奖励 + 出口
   - 创建 Z2d 房间群（3~4 个房间）：前室 + Boss 竞技场 + 奖励 + ACT2 出口
   - Boss 竞技场 size 至少 18x12（需求 4.6）
   - Boss 前走廊不超过连续 3 个线性房间（需求 1.5）
   - boss 类型 + enemy + checkpoint 元素（需求 4.1、4.3）
   - ACT2 出口 door 元素预留（需求 4.4）
   - _需求：1.5, 4.3, 4.6, 5.4, 6.2, 6.3_

- [ ] 10. 编写全部 connections 和 doorLinks，完成最终 JSON 文件
   - 编写所有双向连接（A→B + B→A）对于普通通道（需求 5.7）
   - 编写所有单向连接（仅 A→B）对于捷径/跌落（需求 5.8，至少 2 条）
   - 为每个有 door 元素的房间编写对应的 doorLinks（需求 5.6）
   - 验证 JSON 格式完全对齐 LevelDesigner.html：rooms/connections/doorLinks（需求 5.1~5.6）
   - 验证总房间数在 30~50 之间（需求 6.5）
   - 验证环路数量 ≥ 3（需求 1.3）
   - 验证单向连接 ≥ 2 条（需求 3.3）
   - 验证 floor 使用至少 3 个不同值（需求 2.1）
   - 添加 comment 字段标注每个区域分隔（需求 5.9）
   - 添加 levelName 和 description 顶层字段
   - _需求：1.3, 2.1, 3.3, 5.1~5.9, 6.5_
