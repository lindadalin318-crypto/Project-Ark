# 实施计划：Track 网格强制替换

- [ ] 1. 重构放置逻辑：移除"占用即拒绝"，改为收集被覆盖部件
   - 找到当前 `mouseup` 放置函数中检测格子占用后 `return` 的逻辑
   - 改为：遍历新部件 `shape` 覆盖的所有格子，收集所有 `itemId` 去重，得到 `evictedItems[]`
   - 对每个被顶出的多格部件，清除其在 `grid` 中**所有**占用格子（含未被新部件覆盖的格子）
   - 清除完毕后写入新部件数据，保留越界检测作为唯一拒绝条件
   - _需求：1.1、1.2、1.3、1.4_

- [ ] 2. 实现被顶出部件的库存回收
   - 将 `evictedItems[]` 按从左上到右下的顺序依次 `push` 到库存数组末尾
   - 移除被顶出部件在 Track 网格上的覆盖层 DOM（`overlay` 元素）
   - 恢复被清除格子为空格显示（重新渲染对应格子的 `+` 占位符）
   - 调用库存渲染函数，将新追加的部件渲染到库存网格末尾
   - _需求：3.1、3.2、3.3_

- [ ] 3. 更新拖拽预览高亮：三色状态系统（绿/橙/红）
   - 在 `mousemove` 悬停预览逻辑中，区分三种状态：
     - 越界 → 红色高亮（`drag-invalid`）
     - 有占用但不越界 → 橙色高亮（`drag-replace`），新增对应 CSS 样式
     - 全空 → 绿色高亮（`drag-valid`，已有）
   - 在 Ghost 元素上叠加替换提示文字（`↺ 替换 N 个部件`），占用时显示，全空时隐藏
   - _需求：2.1、2.2、2.3、2.4_

- [ ] 4. 处理 Track 间拖拽与同 Track 内移动的自身冲突
   - 当拖拽来源为 Track（而非库存）时，在执行放置前先临时清除该部件在来源 Track 的占用
   - 若目标与来源为同一 Track，确保自身格子不被计入 `evictedItems`（避免自顶自）
   - 若目标 Track 类型与部件类型不匹配，拒绝放置并恢复来源 Track 数据
   - _需求：4.1、4.2、4.3_

- [ ] 5. 实现部件飞回库存动画
   - 5.1 创建飞行克隆体并执行飞行动画
     - 在顶出部件时，用 `getBoundingClientRect()` 获取覆盖层屏幕坐标，创建 `fixed` 定位的克隆体 DOM
     - 用 `getBoundingClientRect()` 获取库存目标格子坐标，计算位移差值
     - 通过 CSS `transition`（translate + scale + opacity）执行 350ms `ease-in` 飞行动画
     - 多部件顶出时每个克隆体错开 50ms stagger 依次启动
     - _需求：6.1、6.2、6.3、6.4_
   - 5.2 处理动画生命周期与防闪现
     - 飞行期间将库存真实部件 DOM 设为 `opacity: 0`
     - 动画结束后移除克隆体，恢复库存部件可见，并播放落地弹跳（scale 1.2→1.0，150ms）
     - 维护全局 `activeFlightAnimations[]` 列表，新拖拽发起时立即 skip 所有进行中动画
     - _需求：6.5、6.6、6.7_

- [ ] 6. 通知栏消息 & Edge Cases 防护
   - 强制替换成功后在通知栏显示 `REPLACED: [新部件名] ↔ [被替换部件名, ...]`
   - 确认 Escape 键取消拖拽时不执行顶出逻辑，数据不变
   - 确认 `window` 级 `mouseup` 在鼠标移出窗口后正确取消拖拽并回滚数据
   - 替换完成后断言 `grid` 数据与 DOM 覆盖层完全同步（无幽灵占用/幽灵覆盖层）
   - _需求：1.5、5.1、5.2、5.3、5.4_
