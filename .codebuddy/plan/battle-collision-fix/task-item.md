# 实施计划：战斗碰撞修复 + 不规则弹射系统

- [ ] 1. 修正碰撞检测距离
   - 将 `collisionDist` 常量从 `28` 修改为 `38`（两个 radius=18 的陀螺相切距离 36 + 2px 余量）
   - 将碰撞触发判断从硬编码常量改为动态计算 `p.radius + e.radius`，使不同半径的陀螺也能正确碰撞
   - 确认 overlap 分离修正逻辑使用相同的动态距离，保持两陀螺不重叠
   - _需求：1.1、1.2、1.3、1.4_

- [ ] 2. 修正轨道扰动五项参数
   - 将 `orbitDistanceMin` 从 `ARENA.radius * 0.25` 改为 `ARENA.radius * 0.05`（约 11px，覆盖近距离绕圈）
   - 将 `orbitDistanceMax` 从 `ARENA.radius * 0.65` 改为 `ARENA.radius * 0.85`（约 187px，覆盖远距离绕圈）
   - 将 `orbitBreakImpulse` 从 `0.22` 改为 `0.45`（冲量翻倍，确保有效推开）
   - 将 `orbitDetectFrames` 从 `180` 改为 `120`（约 2 秒，更快响应僵局）
   - 将 `orbitBreakCooldown` 从 `300` 改为 `180`（约 3 秒，允许更频繁扰动）
   - _需求：2.1、2.2、2.3、2.4、2.5、2.6_

- [ ] 3. 在 ARENA 常量中添加不规则弹射参数
   - 在 ARENA 常量对象中新增 `irregularDeflectAngle: 20`（单位：度）
   - 确保该参数为 0 时系统退化为原有理想弹性碰撞行为（向下兼容）
   - _需求：3.3、3.4_

- [ ] 4. 实现碰撞后不规则弹射角度偏转逻辑
   - 在碰撞物理冲量计算完成后，对玩家和敌人各自独立生成 `±irregularDeflectAngle` 范围内的随机角度
   - 将随机角度旋转矩阵应用到各自的速度向量（vx/vy）上，不影响 RPM 伤害计算
   - 冷却跳过碰撞时同样应用随机偏转（物理弹射不受冷却影响）
   - _需求：3.1、3.2、3.5、3.6_

- [ ] 5. 更新 BattleLog 记录轨道扰动详情
   - 在轨道扰动触发的日志中追加当前距离、冲量值和稳定帧数三项信息
   - 确认战斗结算日志中的"轨道扰动触发次数"统计正常工作
   - _需求：2.7_
