# 实施计划：陀螺战斗节奏优化

- [ ] 1. 修复弹性系数与碰撞粘连问题
   - 将 `restitution` 调整至 0.9～1.1 的适中范围（当前为 1.2 过高）
   - 确认碰撞后位置强制分离逻辑：中心距 ≥ 碰撞检测距离 + 2px
   - 确认最小分离速度阈值已正确设置，防止低速粘连
   - _需求：3.1、3.2、3.4_

- [ ] 2. 添加碰撞冷却帧机制（防重复计算）
   - 在碰撞处理函数中新增 `collisionCooldownFrames` 计数器（约 10 帧）
   - 冷却期间跳过碰撞检测，冷却结束后重置计数器
   - _需求：3.3_

- [ ] 3. 实现相对速度驱动的碰撞伤害与弹开力
   - 碰撞时计算双方速度向量的相对速度大小
   - 将相对速度映射为伤害倍率（低速 → 低倍率，高速 → 高倍率）
   - 将相对速度映射为弹开冲量大小（低速 → 小弹开，高速 → 大弹开）
   - 保留 ±15°～±25° 随机偏转角模拟菱角不规则弹射
   - _需求：1.1、1.2、1.3、1.4、1.5_

- [ ] 4. 重构轨道扰动方向逻辑（概率混合）
   - 扰动触发时：60%～70% 概率朝向对方（在正对方向叠加 ±45° 随机偏转）
   - 扰动触发时：30%～40% 概率施加完全随机方向冲量
   - 在战斗日志中记录每次扰动的方向类型（朝向/随机）及当前双方距离
   - _需求：2.1、2.2、2.6_

- [ ] 5. 实现距离自适应扰动频率
   - 距离较远（> 碰撞检测距离 × 3）时缩短扰动冷却时间（提高频率）
   - 距离较近（≤ 碰撞检测距离 × 1.5）时延长扰动冷却时间（降低频率）
   - 修复冷却期间稳定帧计数器未重置的 Bug（防止冷却后立即重复触发）
   - _需求：2.3、2.4、2.5_

- [ ] 6. 实现战斗时长软控制机制
   - 战斗开始后追踪"距上次碰撞经过的时间"计时器
   - 战斗开始 5s 内无首次碰撞 → 强制触发一次高强度朝向扰动
   - 任意连续 8s 无碰撞 → 自动触发一次扰动打破僵局
   - 战斗超过 45s 未结束 → 逐步提高扰动频率和冲量强度
   - _需求：4.1、4.2、4.3_

- [ ] 7. 完善战斗结束日志输出
   - 战斗结束时在日志中输出：总碰撞次数、战斗时长、扰动触发次数
   - _需求：4.4_
